LEC 01
Introduction to Data Communications
Components · Characteristics · Network types · Topology
Q1 What are the four main characteristics that determine the effectiveness of a data communication system? Define each. ▾
▸ Solution
1. Delivery — The system must deliver data to the correct destination only. Data must reach the right device and the right user.

2. Accuracy — The system must deliver data accurately, without alteration. Data changed during transmission is unusable.

3. Timeliness — The system must deliver data in a timely manner. Late delivery is useless for real-time applications (audio/video = real-time transmission).

4. Jitter — Variation in the packet arrival time. If the delay for packet 1 = 20 ms, packet 2 = 45 ms, packet 3 = 40 ms — this jitter causes problems for time-sensitive applications.

Q5 For a fully connected mesh topology with n devices: how many physical links are required? How many ports does each device need? Calculate for n = 5. ▾
▸ Solution

For n = 5:

Ports per device = 5 − 1 = 4 ports
Key advantage: Mesh provides dedicated links = no traffic problems, robust, easy to isolate faults. Disadvantage: High cost (cables + ports).
Q6 What is the difference between analog data and digital data? Give one example of each. ▾
▸ Solution
Analog data: Continuous — takes on any value over a range. Example: sound (voice), temperature readings from a clock.

Digital data: Discrete — takes on a limited set of defined values. Example: text stored as ASCII codes, integers in a computer.

Q2 What is encapsulation? Describe what happens at each layer when data is sent from sender to receiver. ▾
▸ Solution
Encapsulation means each layer adds its own header (H) — and sometimes a trailer (T) — to the data it receives from the layer above. The data portion of a packet at level N−1 carries the entire packet from level N and treats it as one integral unit (level N−1 does not see inside).

At the sender (top → down):
• Application (7) → creates the message
• Each layer adds its H (and T at layer 2)
• Physical (1) → converts to bits and sends

At the receiver (bottom → up):
• Each layer strips its own H (and T)
• Layer 7 receives the original message

Result: Layer N on machine A communicates logically with Layer N on machine B (peer-to-peer communication).
Q3 What are the five roles/responsibilities of the Data Link Layer? State the unit of data at this layer. ▾
▸ Solution
Unit: Frame (hop-to-hop delivery)

1. Framing: Divides bit stream from network layer into frames.
2. Physical Addressing: Adds header with sender/receiver physical address (MAC).
3. Flow Control: If receiver is slower than sender, a flow control mechanism prevents overwhelming the receiver.
4. Error Control: Adds mechanisms to detect and retransmit damaged/lost frames; prevents duplicate frames (via trailer).
5. Access Control: When multiple devices share a link, determines which device has control at any given time.

Q5 In the OSI model, which layers are "network support layers" vs "user support layers"? What connects them? ▾
▸ Solution
Network Support Layers (Physical aspects): Layers 1, 2, 3 — deal with physical transmission of data from one device to another.

User Support Layers (Software aspects): Layers 5, 6, 7 — allow interoperability among unrelated software systems.

Layer 4 (Transport) is the link — it ensures that what the lower layers transmitted is in a form usable by the upper layers. It bridges the gap.

Hardware vs Software: Lower layers (1, 2, 3) → hardware + software. Upper layers (4, 5, 6, 7) → software only. Physical layer → mostly hardware.
LEC 03
Data and Signals
Digital signals · Nyquist · Shannon · Attenuation · Performance metrics

⚡ Key Formulas — Data & Signals

Nyquist (noiseless): BitRate = 2 × Bandwidth × log₂L
Shannon (noisy): Capacity = Bandwidth × log₂(1 + SNR)
SNR (dB): SNR_dB = 10 × log₁₀(SNR)
Attenuation (dB): dB = 10 × log₁₀(P₂/P₁)
Bit Length: Bit Length = Propagation Speed × Bit Duration
Propagation Time: Prop. Time = Distance / Propagation Speed
Transmission Time: Trans. Time = Message Size / Bandwidth
Latency: Latency = Prop. Time + Trans. Time + Queuing + Processing
Throughput: Throughput = (frames/sec) × (bits/frame)
Q1 A digital signal has 8 levels. How many bits are needed per level? ▾
▸ Solution
Number of bits = log₂ L = log₂ 8 = 3 bits per level
Each signal level is represented by 3 bits.
Q2 A digital signal has 9 levels. How many bits per level? ▾
▸ Solution
log₂ 9 ≈ 3.17 bits → not an integer, must round up to power of 2 → 4 bits
Answer: 4 bits per level.
Rule: The number of bits must be a whole integer AND a power of 2. So round log₂L up to the next power of 2.
Q3 We need to download text documents at the rate of 100 pages per second. Assume a page has 24 lines with 80 characters each, and each character requires 8 bits. What is the required bit rate? ▾
▸ Solution
Bits per page = 24 lines × 80 chars × 8 bits = 15,360 bits/page Bit rate = 100 pages/s × 15,360 bits/page = 1,536,000 bps ≈ 1.536 Mbps
Q4 (Attenuation) A signal's power is reduced to one-half after passing through a medium. What is the attenuation in dB? ▾
▸ Solution
dB = 10 × log₁₀(P₂/P₁) = 10 × log₁₀(1/2) = 10 × (−0.3) = −3 dB
A loss of −3 dB is equivalent to losing one-half the power.
Q5 A signal is amplified so its power increases 10 times. What is the gain in dB? ▾
▸ Solution
dB = 10 × log₁₀(P₂/P₁) = 10 × log₁₀(10) = 10 × 1 = +10 dB
Q6 A signal travels through three cascaded links with dB values of −3 dB, +7 dB, and −5 dB. What is the total dB change? ▾
▸ Solution
Total dB = (−3) + (+7) + (−5) = −1 dB
Key concept: Decibel values are simply added (or subtracted) when cascading — that is why engineers use dB!
Q7 The power of a signal is given as dBm = −30. Calculate the power in milliwatts. ▾
▸ Solution
dBm = 10 × log₁₀(Pm) → −30 = 10 × log₁₀(Pm) log₁₀(Pm) = −3 → Pm = 10^(−3) = 0.001 mW
Q8 A cable has a loss of −0.3 dB/km. If the starting power is 2 mW, what is the power at 5 km? ▾
▸ Solution
Loss = 5 × (−0.3) = −1.5 dB −1.5 = 10 × log₁₀(P₂/2) log₁₀(P₂/2) = −0.15 P₂/2 = 10^(−0.15) ≈ 0.708 P₂ = 2 × 0.708 ≈ 1.416 mW
Q9 ★ (Nyquist) A noiseless channel has a bandwidth of 3000 Hz and uses two signal levels. What is the maximum bit rate? ▾
▸ Solution
BitRate = 2 × B × log₂ L = 2 × 3000 × log₂ 2 = 2 × 3000 × 1 = 6000 bps
Q10 ★ (Nyquist) Same noiseless channel, 3000 Hz bandwidth, but now with 4 signal levels. What is the maximum bit rate? ▾
▸ Solution
BitRate = 2 × 3000 × log₂ 4 = 6000 × 2 = 12,000 bps
Note: Increasing signal levels increases bit rate, but also increases reliability problems. There is a practical limit.
Q11 ★ (Nyquist) We need to send 265 kbps over a noiseless channel with a bandwidth of 20 kHz. How many signal levels do we need? ▾
▸ Solution
265,000 = 2 × 20,000 × log₂ L log₂ L = 265,000 / 40,000 = 6.625 L = 2^6.625 ≈ 98.7 → not a power of 2
Since 98.7 is not a power of 2:
• If L = 128: BitRate = 2 × 20,000 × 7 = 280 kbps
• If L = 64: BitRate = 2 × 20,000 × 6 = 240 kbps
We either increase levels to 128 or reduce bit rate to 240 kbps.
Q12 ★ (Shannon) A telephone line has a bandwidth of 3000 Hz and SNR = 3162. What is the maximum channel capacity? ▾
▸ Solution
C = B × log₂(1 + SNR) = 3000 × log₂(1 + 3162) = 3000 × log₂(3163) = 3000 × 11.62 ≈ 34,860 bps ≈ 34.86 kbps
The theoretical maximum bit rate of this telephone line is 34.86 kbps.
Q13 ★ (Shannon) SNR_dB = 36 and bandwidth = 2 MHz. Calculate the theoretical channel capacity. ▾
▸ Solution
SNR_dB = 10 × log₁₀(SNR) 36 = 10 × log₁₀(SNR) → SNR = 10^3.6 ≈ 3981 C = 2,000,000 × log₂(1 + 3981) = 2,000,000 × log₂(3982) = 2,000,000 × 11.96 ≈ 23.9 Mbps
Q14 ★ (Both Formulas) A channel has 1 MHz bandwidth and SNR = 63. Find the appropriate bit rate and number of signal levels. ▾
▸ Solution
Step 1: Use Shannon to find the upper limit:
C = 1,000,000 × log₂(1 + 63) = 1,000,000 × log₂(64) = 1,000,000 × 6 = 6 Mbps
Step 2: Choose a lower practical rate, e.g., 4 Mbps. Then use Nyquist to find L:
4,000,000 = 2 × 1,000,000 × log₂ L log₂ L = 2 → L = 4 signal levels
Rule: Shannon gives the upper limit; Nyquist tells us how many signal levels we need to achieve a chosen rate below that limit.
Q15 ★ (Propagation Time) Distance = 12,000 km, propagation speed = 2.4 × 10⁸ m/s. Find the propagation time. ▾
▸ Solution
Propagation Time = Distance / Speed = 12,000,000 m / (2.4 × 10⁸ m/s) = 0.05 s = 50 ms
A bit can cross the Atlantic Ocean in only 50 ms.
Q16 ★ (Latency) A 2.5-kbyte email is sent over a 1 Gbps network. Distance = 12,000 km, light speed = 2.4 × 10⁸ m/s. Find propagation time and transmission time. ▾
▸ Solution
Propagation Time = 12,000,000 / (2.4 × 10⁸) = 0.05 s = 50 ms Message size = 2.5 × 1024 × 8 = 20,480 bits Transmission Time = 20,480 / (1 × 10⁹) ≈ 0.00002 s = 0.02 ms
Conclusion: For a short message over a high-bandwidth link, propagation time (50 ms) dominates. Transmission time (0.02 ms) is negligible.
Q17 ★ (Throughput) A network with 10 Mbps bandwidth passes 12,000 frames/minute, each frame carrying 10,000 bits. What is the throughput? ▾
▸ Solution
Throughput = (12,000 frames/min) × (10,000 bits/frame) ÷ 60 s/min = 120,000,000 / 60 = 2,000,000 bps = 2 Mbps
Throughput (2 Mbps) is about one-fifth of the bandwidth (10 Mbps).
Key concept: Throughput ≤ Bandwidth. Actual (throughput) is always less than potential (bandwidth).

Q1 What is segmentation? What are its advantages and disadvantages? ▾
▸ Solution
Segmentation: At transmission, a message is divided into smaller transmittable segments, each with a sequence number. On receiving, reassembly reorders them back to the original message.

AdvantagesDisadvantages

Shorter delaysMore processing time needed

Q2 What is Frequency Division Multiplexing (FDM)? Describe the four steps of the FDM process. ▾
▸ Solution
FDM is an analog multiplexing technique that combines analog signals by dividing the total bandwidth into sub-channels, each at a different frequency.

FDM Process:
1. Divide the available bandwidth into independent frequency channels (with guard bands between them to prevent interference).
2. Modulate each signal to a different frequency band.
3. Combine all modulated signals into one composite signal using a multiplexer.
4. At the receiver, decompose the composite signal using a demultiplexer and filters (demodulate each sub-signal).

Guard bands: Unused frequency gaps between channels to prevent interference. They waste some capacity.
Q3 ★ (FDM) Three voice channels, each 4 kHz, are to be combined on a link with bandwidth from 20 to 32 kHz. Show how, assuming no guard bands. ▾
▸ Solution
Total available bandwidth = 32 − 20 = 12 kHz. Three channels × 4 kHz = 12 kHz ✓
Channel 1 → 20 to 24 kHz Channel 2 → 24 to 28 kHz Channel 3 → 28 to 32 kHz
Each voice channel is shifted (modulated) to its own frequency band, then combined as a composite signal.
Q4 ★ (FDM) Five channels, each 100 kHz, need to be multiplexed together with guard bands of 10 kHz between channels. What is the minimum link bandwidth? ▾
▸ Solution
For 5 channels, we need 4 guard bands (between consecutive channels).
Minimum bandwidth = (5 × 100 kHz) + (4 × 10 kHz) = 500 + 40 = 540 kHz
Q5 ★ (FDM / AMPS) The AMPS cellular system uses a band from 824–849 MHz for sending (25 MHz). Each user has 30 kHz bandwidth. How many people can simultaneously use cellular phones? ▾
▸ Solution
Total channels = 25 MHz / 30 kHz = 25,000 / 30 ≈ 833 Control channels (used for signaling) = 42 Available voice channels = 833 − 42 − (rounding) = 790 channels
790 simultaneous users can use cellular phones.
Q6 ★ (Synchronous TDM) Three 1-kbps connections are multiplexed. One bit per unit. Find: (a) duration of each input slot, (b) duration of each output slot, (c) duration of each frame. ▾
▸ Solution
a. Input bit duration = 1 / 1 kbps = 1 ms Input slot duration = 1 ms b. Output slot = Input slot / n = 1 ms / 3 = 1/3 ms ≈ 0.333 ms c. Frame = n × output slots = 3 × (1/3 ms) = 1 ms (Frame duration = input unit duration)
Rule: Frame duration always equals input unit duration. Output slot = input slot / n.
Q7 ★ (Synchronous TDM) Four 1-Mbps connections are multiplexed. Find: (a) input bit duration, (b) output bit duration, (c) output bit rate, (d) output frame rate. ▾
▸ Solution
n = 4 connections a. Input bit duration = 1 / 1 Mbps = 1 μs b. Output bit duration = 1 μs / 4 = 0.25 μs = ¼ μs c. Output bit rate = 1 / (0.25 μs) = 4 Mbps (= 4 × input rate = 4 × 1 Mbps = 4 Mbps ✓) d. Frame rate = input bit rate = 1,000,000 frames/second Verify: 1,000,000 frames/s × 4 bits/frame = 4 Mbps ✓
Q8 ★ (Synchronous TDM) Four 1-kbps connections multiplexed, 1 bit per unit. Find: (a) bit duration before MUX, (b) link transmission rate, (c) time slot duration, (d) frame duration. ▾
▸ Solution
a. Bit duration before MUX = 1 / 1 kbps = 1 ms b. Link rate = 4 × 1 kbps = 4 kbps c. Time slot duration = 1 ms / 4 = 0.25 ms = 250 μs (also = 1 / 4 kbps = 250 μs ✓) d. Frame duration = 4 × 250 μs = 1 ms (= input unit duration ✓)

Q6 What is the difference between step-index and graded-index multimode fiber? ▾
▸ Solution
Step-index: Uniform refractive index in the core; a sharp, sudden decrease at the core-cladding interface. Light bounces off the cladding at sharp angles. Used in both single-mode and some multimode fibers.

Graded-index: Refractive index decreases continuously (parabolically) from the center of the core outward. Light rays follow sinusoidal (curved) paths. This minimizes modal dispersion compared to step-index multimode fiber.
Result: Graded-index fiber has less distortion than step-index multimode fiber, allowing better performance over longer distances.
LEC 06
Error Detection and Correction
Single-bit vs Burst errors · Checksum · One's complement
Q1 What is the difference between a single-bit error and a burst error? Which is more common in serial transmission? ▾
▸ Solution
Single-bit error: Only 1 bit in the data unit has changed (0 → 1 or 1 → 0). Rare in serial transmission because consecutive bits are very close together.

Burst error: Two or more bits in the data unit have changed. The length of a burst error is measured from the first corrupted bit to the last. More common in serial transmission because noise usually affects a period of time, corrupting multiple consecutive bits.

Note: Burst error length ≠ number of corrupted bits. It is the span from first to last corrupted bit (bits in between may or may not be corrupted).
Q2 Distinguish between error detection and error correction. Which requires more redundant bits? ▾
▸ Solution
Error Detection: We only check whether any error has occurred (yes/no). Simpler; fewer redundant bits needed.

Error Correction: We must know the exact number of corrupted bits AND their location so we can fix them. Requires more redundant bits.

Redundancy: Extra bits added by the sender that create a mathematical relationship with the data bits. The receiver checks this relationship to detect or correct errors.
Q3 Explain the Internet Checksum process at both the sender and receiver sides. ▾
▸ Solution
At the Sender:
1. Divide the message into 16-bit words.
2. Set the checksum word to 0.
3. Add all words (including the 0 checksum) using one's complement addition.
4. Complement the sum → this becomes the checksum.
5. Send the data + checksum.

At the Receiver:
1. Divide the received message (including checksum) into 16-bit words.
2. Add all words using one's complement addition.
3. Complement the result.
4. If the result = 0000 0000 0000 0000 → no error; accept.
If result ≠ 0 → error; reject.
Q4 (One's Complement) Represent the number 21 using one's complement arithmetic with only 4 bits. ▾
▸ Solution
21 in binary = 10101 (needs 5 bits). With only 4 bits, we "wrap around" the leftmost bit:
10101 → wrap leftmost bit: 0101 + 1 (the wrapped bit) = 0110 = 6
Answer: 6 (21 mod 15 in one's complement 4-bit representation).
Q5 (One's Complement) Represent −6 in 4-bit one's complement. ▾
▸ Solution
+6 in 4-bit binary = 0110 Negative (complement) = invert all bits = 1001 = 9
So −6 is represented as 1001 (= 9 in unsigned). The complement of 6 is 9 in 4-bit one's complement.
Rule: In one's complement, the negative of a number is found by inverting ALL bits (0 ↔ 1).
Q6 ★ (Full Checksum Calculation) Calculate the Internet Checksum for the text "Forouzan" (8 characters). Use ASCII and show work for both sender and receiver. ▾
▸ Solution
The text "Forouzan" = 8 bytes = 4 words of 16 bits each:
Word 1: Fo = 0100 0110 0110 1111 (46 6F) Word 2: ro = 0111 0010 0110 1111 (72 6F) Word 3: uz = 0111 0101 0111 1010 (75 7A) Word 4: an = 0110 0001 0110 1110 (61 6E) Sender: Checksum (initial) = 0000 0000 0000 0000 Sum of all 5 words = 1000 1111 1100 0111 (8F C7) One's complement = 0111 0000 0011 1000 (70 38) ← CHECKSUM SENT Receiver: Receives: Fo, ro, uz, an + checksum (70 38) Sum = 1111 1111 1111 1111 (FF FF) One's complement = 0000 0000 0000 0000 → ALL ZEROS = NO ERROR ✓
Q7 What is the basic idea behind the checksum (simple version)? How does the receiver use it? ▾
▸ Solution
Sender: Sends data (e.g., 7, 11, 12, 0, 6) along with their sum (36) or the negative of the sum (−36).

With negative checksum (−36): Receiver adds all numbers including the checksum: 7 + 11 + 12 + 0 + 6 + (−36) = 0. If the result = 0 → no error. If ≠ 0 → error.
Sending the negative of the sum as the checksum simplifies the receiver's job — it just adds everything up and checks for 0.

Q2 ★ How can you determine whether a MAC address is unicast, multicast, or broadcast? For each address below, state the type: (a) 4A:30:10:21:10:1A, (b) 47:20:1B:2E:08:EE, (c) FF:FF:FF:FF:FF:FF ▾
▸ Solution
Rule: Look at the least significant bit (LSB) of the first byte.
  • If bit = 0 (even hex digit) → Unicast
  • If bit = 1 (odd hex digit) → Multicast
  • If all bits are 1 (FF:FF:FF:FF:FF:FF) → Broadcast

(a) 4A:30:10:21:10:1A
First byte = 4A → "A" in binary = 1010 → LSB = 0 (even) → Unicast

(b) 47:20:1B:2E:08:EE
First byte = 47 → "7" in binary = 0111 → LSB = 1 (odd) → Multicast

(c) FF:FF:FF:FF:FF:FF
All bits = 1 → Broadcast (special case of multicast)
Q3 ★ Show how the MAC address 47:20:1B:2E:08:EE is sent on the line (bit transmission order). ▾
▸ Solution
Addresses are sent byte by byte from left to right, but within each byte, bits are sent right to left (LSB first):
47 = 0100 0111 → sent as: 1 1 1 0 0 0 1 0 20 = 0010 0000 → sent as: 0 0 0 0 0 1 0 0 1B = 0001 1011 → sent as: 1 1 0 1 1 0 0 0 2E = 0010 1110 → sent as: 0 1 1 1 0 1 0 0 08 = 0000 1000 → sent as: 0 0 0 1 0 0 0 0 EE = 1110 1110 → sent as: 0 1 1 1 0 1 1 1
Q4 Explain the CSMA/CD protocol and how it handles collisions. ▾
▸ Solution
CSMA/CD = Carrier Sense Multiple Access / Collision Detection

How it works:
1. A station wants to send → it listens (senses carrier) to check if the medium is idle.
2. If the medium is idle → it transmits.
3. While transmitting → it monitors the medium for a collision.
4. If a collision is detected → both stations immediately stop transmitting.
5. Each station waits a random backoff time before attempting to retransmit.
6. This reduces the probability of another collision.

Only one station can transmit at a time on a shared medium. CSMA/CD is used in half-duplex Ethernet (with hub). Full-duplex (with switch) does not need CSMA/CD.

Q8 What is a bridge in Ethernet? How does it improve performance? ▾
▸ Solution
A bridge connects two or more LAN segments, dividing a large Ethernet network into separate collision domains. Each segment has fewer devices, so collisions are less frequent.

When to use a bridge:
1. When a large number of new users join the network.
2. When applications requiring high bandwidth (databases, video conferencing) are added.

Result: Each segment gets its own dedicated bandwidth; overall network performance improves because broadcasts and collisions are contained within each segment.
