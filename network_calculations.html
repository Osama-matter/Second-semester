LEC 03 Data & Signals — Calculations
3.1 A digital signal has 8 levels. How many bits are needed per level? ▾
▸ Solution
Bits per level = log₂ L = log₂ 8 = 3 bits
Answer: 3 bits per level
3.2 A digital signal has 9 levels. How many bits per level? ▾
▸ Solution
log₂ 9 ≈ 3.17 → not an integer power of 2 Round UP to next power of 2 → 4 bits
⚠ The result must be a whole number AND a power of 2. Always round up.
Answer: 4 bits per level
3.3 Download 100 pages/second. Each page = 24 lines × 80 chars, each char = 8 bits. What is the required bit rate? ▾
▸ Solution
Bits per page = 24 × 80 × 8 = 15,360 bits Bit rate = 100 pages/s × 15,360 bits/page = 1,536,000 bps ≈ 1.536 Mbps
Answer: 1.536 Mbps
3.4 (Attenuation) Signal power reduced to one-half. Find attenuation in dB. ▾
▸ Solution — Example 3.26
dB = 10 · log₁₀(P₂/P₁) = 10 · log₁₀(1/2) = 10 · (−0.301) = −3 dB
📌 −3 dB always means power is halved. +3 dB means power is doubled.
Answer: −3 dB
3.5 (Amplification) Signal power increased 10 times. Find gain in dB. ▾
▸ Solution — Example 3.27
dB = 10 · log₁₀(P₂/P₁) = 10 · log₁₀(10) = 10 · 1 = +10 dB
Answer: +10 dB
3.6 (Cascading) Signal passes 3 points: −3 dB → +7 dB → −5 dB. Total dB? ▾
▸ Solution — Example 3.28
Total dB = (−3) + (+7) + (−5) = −1 dB
📌 dB values are simply ADDED when cascading — that is why engineers use decibels!
Answer: −1 dB (net loss)
3.7 (dBm) Signal has dBm = −30. Find power in milliwatts. ▾
▸ Solution — Example 3.29
dBm = 10 · log₁₀(Pm) −30 = 10 · log₁₀(Pm) log₁₀(Pm) = −3 Pm = 10^(−3) = 0.001 mW
Answer: 0.001 mW (= 1 μW)
3.8 (Cable loss) Loss = −0.3 dB/km, starting power = 2 mW. Find power at 5 km. ▾
▸ Solution — Example 3.30
Step 1: Total loss = 5 × (−0.3) = −1.5 dB Step 2: −1.5 = 10 · log₁₀(P₂/2) log₁₀(P₂/2) = −0.15 P₂/2 = 10^(−0.15) ≈ 0.708 P₂ = 2 × 0.708 ≈ 1.416 mW
Answer: ≈ 1.416 mW
3.9 ★ (Nyquist) Noiseless channel: BW = 3000 Hz, signal levels L = 2. Max bit rate? ▾
▸ Solution — Example 3.24
BitRate = 2 × B × log₂ L = 2 × 3000 × log₂ 2 = 6000 × 1 = 6,000 bps
Answer: 6,000 bps
3.10 ★ (Nyquist) Same channel BW = 3000 Hz, now L = 4 levels. Max bit rate? ▾
▸ Solution — Example 3.25
BitRate = 2 × 3000 × log₂ 4 = 6000 × 2 = 12,000 bps
Answer: 12,000 bps
3.11 ★ (Nyquist — find L) Need to send 265 kbps over a noiseless channel with BW = 20 kHz. How many signal levels? ▾
▸ Solution — Example 3.26
265,000 = 2 × 20,000 × log₂ L log₂ L = 265,000 / 40,000 = 6.625 L = 2^6.625 ≈ 98.7 ← not a power of 2 Options: If L = 128 → BR = 2 × 20,000 × 7 = 280 kbps (faster) If L = 64 → BR = 2 × 20,000 × 6 = 240 kbps (slower)
📌 Since 98.7 is not a power of 2, we round to the nearest power: 64 or 128 and accept the resulting bit rate.
Answer: L=128 → 280 kbps OR L=64 → 240 kbps
3.12 ★ (Shannon — SNR=0) An extremely noisy channel (SNR ≈ 0). What is the capacity? ▾
▸ Solution — Example 3.27
C = B × log₂(1 + SNR) = B × log₂(1 + 0) = B × log₂(1) = B × 0 = 0 bps
📌 Capacity = 0 regardless of bandwidth. You cannot send data through pure noise.
Answer: 0 bps
3.13 ★ (Shannon — telephone) BW = 3000 Hz, SNR = 3162. Max capacity? ▾
▸ Solution — Example 3.28
C = 3000 × log₂(1 + 3162) = 3000 × log₂(3163) = 3000 × 11.62 ≈ 34,860 bps ≈ 34.86 kbps
Answer: ≈ 34.86 kbps
3.14 ★ (Shannon — dB SNR) SNRdB = 36, BW = 2 MHz. Find theoretical capacity. ▾
▸ Solution — Example 3.29
Step 1: Convert dB to linear 36 = 10 · log₁₀(SNR) log₁₀(SNR) = 3.6 SNR = 10^3.6 ≈ 3981 Step 2: Apply Shannon C = 2,000,000 × log₂(1 + 3981) = 2,000,000 × log₂(3982) = 2,000,000 × 11.96 ≈ 23.9 Mbps
Answer: ≈ 23.9 Mbps
3.15 ★★ (Both formulas) BW = 1 MHz, SNR = 63. Find appropriate bit rate AND number of signal levels. ▾
▸ Solution — Example 3.41
1Use Shannon for upper limit: C = 1,000,000 × log₂(1+63) = 1,000,000 × log₂(64) = 1,000,000 × 6 = 6 Mbps
2Choose a practical rate below 6 Mbps, e.g. 4 Mbps
3Use Nyquist to find L: 4,000,000 = 2 × 1,000,000 × log₂ L → log₂ L = 2 → L = 4 levels
Shannon upper limit = 6 Mbps Chosen rate = 4 Mbps Signal levels (L) = 4
📌 Shannon gives the CEILING. Nyquist tells you HOW MANY LEVELS to achieve your chosen rate.
Answer: 4 Mbps with 4 signal levels
3.16 (Bandwidth × 2) Subscriber line BW = 4 kHz → 56 kbps. If BW doubled to 8 kHz, new rate? ▾
▸ Solution — Examples 3.42 & 3.43
4 kHz BW → 56,000 bps (given) 8 kHz BW → 2 × 56,000 = 112,000 bps
Answer: 112,000 bps (112 kbps)
3.17 ★ (Throughput) BW = 10 Mbps. Network passes 12,000 frames/min, each frame = 10,000 bits. Throughput? ▾
▸ Solution — Example 3.44
Throughput = (12,000 frames/min × 10,000 bits/frame) ÷ 60 s/min = 120,000,000 ÷ 60 = 2,000,000 bps = 2 Mbps
📌 Throughput (2 Mbps) ≈ one-fifth of bandwidth (10 Mbps). Always: Throughput ≤ Bandwidth.
Answer: 2 Mbps
3.18 ★ (Propagation Time) Distance = 12,000 km, speed = 2.4 × 10⁸ m/s. Propagation time? ▾
▸ Solution — Example 3.45
Propagation Time = Distance / Speed = 12,000,000 m / (2.4 × 10⁸ m/s) = 0.05 s = 50 ms
📌 A bit can cross the Atlantic Ocean in just 50 ms!
Answer: 50 ms
3.19 ★ (Latency — short message) Email = 2.5 kbytes, BW = 1 Gbps, distance = 12,000 km, speed = 2.4×10⁸ m/s. Find propagation and transmission times. ▾
▸ Solution — Example 3.46
Propagation Time = 12,000,000 / (2.4 × 10⁸) = 0.05 s = 50 ms Message size = 2.5 × 1024 × 8 = 20,480 bits Transmission Time = 20,480 / (1 × 10⁹) = 0.00002 s = 0.02 ms
📌 Short message + high BW → Propagation time (50 ms) dominates. Transmission time (0.02 ms) is negligible.
Prop = 50 ms · Trans = 0.02 ms
3.20 ★ (Latency — large message) Image = 5 Mbytes, BW = 1 Mbps, distance = 12,000 km, speed = 2.4×10⁸ m/s. Find both times. ▾
▸ Solution — Example 3.47
Propagation Time = 12,000,000 / (2.4 × 10⁸) = 0.05 s = 50 ms Message size = 5 × 1024 × 1024 × 8 = 41,943,040 bits Transmission Time = 41,943,040 / (1 × 10⁶) ≈ 41.94 s
📌 Large message + low BW → Transmission time (41.94 s) dominates. Propagation time (50 ms) is negligible.
Prop = 50 ms · Trans ≈ 41.94 s
LEC 04 Multiplexing — FDM & TDM Calculations
4.1 ★ (FDM) Three voice channels each 4 kHz wide. Link bandwidth = 20–32 kHz. No guard bands. Show channel allocation. ▾
▸ Solution — Example 4.1
Available BW = 32 − 20 = 12 kHz 3 channels × 4 kHz = 12 kHz ✓ Channel 1 → 20 kHz to 24 kHz Channel 2 → 24 kHz to 28 kHz Channel 3 → 28 kHz to 32 kHz
Each channel gets 4 kHz in its own frequency band
4.2 ★ (FDM) 5 channels, each 100 kHz. Guard band = 10 kHz between each pair. Minimum link bandwidth? ▾
▸ Solution — Example 4.2
5 channels → 4 guard bands (between consecutive channels) Min BW = (5 × 100 kHz) + (4 × 10 kHz) = 500 + 40 = 540 kHz
📌 Number of guard bands = number of channels − 1
Answer: 540 kHz
4.3 ★ (AMPS Cellular) Band = 824–849 MHz (25 MHz wide). Each user = 30 kHz. 42 control channels. How many simultaneous users? ▾
▸ Solution — Example 4.4
Total channels = 25 MHz / 30 kHz = 25,000 kHz / 30 kHz ≈ 833 Control channels = 42 Voice channels = 833 − 42 − 1 (rounding) = 790
Answer: 790 simultaneous users
4.4 ★ (Sync TDM) Three 1-kbps connections multiplexed. Unit = 1 bit. Find: (a) input slot, (b) output slot, (c) frame duration. ▾
▸ Solution — Example 4.5
n = 3 connections, each at 1 kbps a. Input bit duration = 1 / 1000 = 1 ms Input slot = 1 ms b. Output slot = Input slot / n = 1 ms / 3 = 1/3 ms ≈ 0.333 ms c. Frame = n × output slots = 3 × (1/3 ms) = 1 ms
📌 Rule: Frame duration = Input slot duration (always equal)
Input: 1 ms · Output: 0.333 ms · Frame: 1 ms
4.5 ★ (Sync TDM) Four 1-Mbps connections multiplexed. Unit = 1 bit. Find: (a) input bit duration, (b) output bit duration, (c) output bit rate, (d) frame rate. ▾
▸ Solution — Example 4.6
n = 4 connections, each at 1 Mbps a. Input bit duration = 1 / 1 Mbps = 1 μs b. Output bit duration = 1 μs / 4 = 0.25 μs (= ¼ μs) c. Output bit rate = 1 / 0.25 μs = 4 Mbps (also = 4 × 1 Mbps = 4 Mbps ✓) d. Frame rate = same as any input rate = 1,000,000 frames/s Verify: 1,000,000 frames/s × 4 bits/frame = 4 Mbps ✓
In: 1 μs · Out: 0.25 μs · Rate: 4 Mbps · Frames: 1M/s
4.6 ★ (Sync TDM) Four 1-kbps connections multiplexed. Unit = 1 bit. Find: (a) bit duration before MUX, (b) link rate, (c) time slot duration, (d) frame duration. ▾
▸ Solution — Example 4.7
n = 4 connections, each at 1 kbps a. Bit duration before MUX = 1 / 1 kbps = 1 ms b. Link rate = 4 × 1 kbps = 4 kbps c. Time slot = 1 ms / 4 = 0.25 ms = 250 μs (also = 1 / 4 kbps = 250 μs ✓) d. Frame duration = 4 × 250 μs = 1 ms (= input bit duration ✓)
Rate: 4 kbps · Slot: 250 μs · Frame: 1 ms
LEC 06 Error Detection — Checksum Calculations
6.1 (Basic Checksum) Sender has data: (7, 11, 12, 0, 6). Explain how the checksum works with positive sum and with negative sum. ▾
▸ Solution — Examples 6.18 & 6.19
Sum = 7 + 11 + 12 + 0 + 6 = 36 Method A (send positive sum): Sender sends: (7, 11, 12, 0, 6, 36) Receiver adds: 7+11+12+0+6 = 36 → matches → NO ERROR ✓ Method B (send negative sum = checksum): Sender sends: (7, 11, 12, 0, 6, −36) Receiver adds ALL: 7+11+12+0+6+(−36) = 0 → result is 0 → NO ERROR ✓ If result ≠ 0 → ERROR!
📌 Method B is preferred: receiver just checks if total = 0.
6.2 ★ (One's Complement) Represent 21 in 4-bit one's complement arithmetic. ▾
▸ Solution — Example 6.20
21 in binary = 10101 (needs 5 bits, too many!) Wrap the leftmost bit (carry wrap-around): 10101 ↓ split into 4-bit + leftover 0101 + 1 (the wrapped bit) = 0110 = 6
Answer: 21 → 0110 (= 6) in 4-bit one's complement
6.3 ★ (One's Complement) Represent −6 in 4-bit one's complement. ▾
▸ Solution — Example 6.21
+6 in 4-bit binary = 0110 To negate: INVERT ALL BITS (0 ↔ 1) 0110 → 1001 = 9
📌 In one's complement: −6 is represented as 1001 (= 9 unsigned). The complement of 6 is 9.
Answer: −6 → 1001 (= 9) in 4-bit one's complement
6.4 ★★ (Internet Checksum) Calculate the checksum for the text "Forouzan" (8 characters). Show sender and receiver sides. ▾
▸ Solution — Example 6.23
1Convert each character to ASCII hex (2 bytes each, group into 16-bit words)
AT THE SENDER: ────────────────────────────────────────── Word Binary Hex Fo = 0100 0110 0110 1111 = 46 6F ro = 0111 0010 0110 1111 = 72 6F uz = 0111 0101 0111 1010 = 75 7A an = 0110 0001 0110 1110 = 61 6E ────────────────────────────────────────── ChSum 0000 0000 0000 0000 = 00 00 ← start with 0 SUM = 1000 1111 1100 0111 = 8F C7 1's C= 0111 0000 0011 1000 = 70 38 ← CHECKSUM SENT AT THE RECEIVER: ────────────────────────────────────────── Fo + ro + uz + an + Checksum(70 38) SUM = 1111 1111 1111 1111 = FF FF 1's C= 0000 0000 0000 0000 = 00 00 ← ALL ZEROS = NO ERROR ✓
Checksum = 0111 0000 0011 1000 (70 38 in hex)
LEC 07 Ethernet — Address & Frame Calculations
7.1 ★ (MAC Address Type) Identify the type of each address:
a) 4A:30:10:21:10:1A  ·  b) 47:20:1B:2E:08:EE  ·  c) FF:FF:FF:FF:FF:FF ▾
▸ Solution — Example 7.1
RULE: Check the LSB (least significant bit) of the FIRST byte. LSB = 0 (even last hex digit) → UNICAST LSB = 1 (odd last hex digit) → MULTICAST All bits = 1 (FF:FF:FF:FF:FF:FF) → BROADCAST a. 4A:30:... → first byte = 4A 'A' in binary = 1010 → LSB = 0 (even) → UNICAST b. 47:20:... → first byte = 47 '7' in binary = 0111 → LSB = 1 (odd) → MULTICAST c. FF:FF:FF:FF:FF:FF → all 1s → BROADCAST
a=Unicast · b=Multicast · c=Broadcast
7.2 ★ (Bit Transmission) Show how 47:20:1B:2E:08:EE is sent on the line, bit by bit. ▾
▸ Solution — Example 7.2
RULE: Bytes sent LEFT → RIGHT Within each byte: bits sent RIGHT → LEFT (LSB first) 47 = 0100 0111 → sent: 1 1 1 0 0 0 1 0 20 = 0010 0000 → sent: 0 0 0 0 0 1 0 0 1B = 0001 1011 → sent: 1 1 0 1 1 0 0 0 2E = 0010 1110 → sent: 0 1 1 1 0 1 0 0 08 = 0000 1000 → sent: 0 0 0 1 0 0 0 0 EE = 1110 1110 → sent: 0 1 1 1 0 1 1 1 Full bit stream: 11100010 00000100 11011000 01110100 00010000 01110111
📌 Each byte's bits are reversed before sending (LSB first). Bytes themselves go in the normal left-to-right order.
7.3 (Frame Size) What is the minimum and maximum size of an Ethernet 802.3 MAC frame? Show the calculation. ▾
▸ Solution
Overhead (fixed fields): Preamble = 7 bytes SFD = 1 byte Destination MAC = 6 bytes Source MAC = 6 bytes Length/Type = 2 bytes CRC = 4 bytes Total overhead = 26 bytes Data field: 46 bytes (min) to 1500 bytes (max) Minimum frame = 26 + 46 = 72 bytes (= 64 bytes without preamble+SFD) Maximum frame = 26 + 1500 = 1526 bytes (= 1518 bytes without preamble+SFD)
Min: 64 bytes · Max: 1518 bytes (excl. preamble)
7.4 ★ (Naming) What does 10Base5 mean? Decode each part. Also explain 10Base2, 10BaseT, 10BaseF. ▾
▸ Solution
Format: [Speed] [Transmission] [Segment] 10 Base 5 Part 1 "10" → Data rate = 10 Mbps Part 2 "Base" → Baseband transmission Part 3 "5" → Max segment = 5 × 100 = 500 m ───────────────────────────────────────────────── Name Speed Cable Type Max Segment ───────────────────────────────────────────────── 10Base5 10 Mbps Thick coax 500 m 10Base2 10 Mbps Thin coax 200 m 10BaseT 10 Mbps Twisted Pair (T=TP) not in name 10BaseF 10 Mbps Fiber Optic (F=FO) not in name ─────────────────────────────────────────────────
10Base5 = 10 Mbps, Baseband, 500m max
