Introduction to PHP & Scripts

What PHP is, how it runs, and the basics of writing PHP syntax

Key Concepts
  • PHP = HyperText Preprocessor — server-side scripting language
  • Code executes on the server; browser receives plain HTML output
  • Files use .php extension; can mix HTML, CSS, JS, PHP
  • PHP tags: <?php … ?>
  • Each statement ends with a semicolon ;
  • Used by ~78% of all websites (Facebook, Wikipedia, WordPress)
  • Frameworks: Laravel, Symfony, Laminas, CodeIgniter, CakePHP
  • PHP learning split: Language → Databases → Security
PHP Can Do
  • Generate dynamic page content
  • Create, read, write, delete files on server
  • Collect & process form data
  • Send and receive cookies
  • Add/modify/delete database records
  • Control user access & encrypt data
  • Output HTML, images, PDFs, XML, XHTML

⚠ Case SensitivityKeywords (echo, if) and functions are NOT case-sensitive. But variable names ARE case-sensitive — $color, $COLOR, and $coLOR are three different variables!

Basic Script Structure
<html>
<body>
<h1>My page</h1>
<?php
  // Single-line comment
  # Also single-line
  /*
    Multi-line comment
  */
  echo "Hello World!"; // semicolon required
  ECHO "Also valid!";  // keywords are case-insensitive
?>
</body>
</html>

ℹ Server-Side AdvantagePHP code is hidden from clients — only the resulting HTML is sent. This enables secure access to databases, files, and server resources without exposing logic.

Variable Rules
  • Starts with $ sign: $myVar
  • No declaration needed — created on first assignment
  • PHP is loosely typed — type can change
  • Constants: define("NAME", value) — no $, global, unchangeable
$x = 42;       // integer
echo $x + 1;   // 43, $x still 42
$x = 'hello';  // type changes — valid!
define("PI", 3.14); echo PI;
Variable Scope
Scope Description
Local Declared inside function — only accessible there
Global Declared outside function — use global $x inside
Static Local but persists after function ends — static $x 
$x = 5;  // global
function myTest() {
  global $x, $y;  // must declare
  $y = $x + 10;   // now works
}
function counter() {
  static $n = 0;  // persists each call
  echo $n++;
}

== vs === (Critical!)
  • == — equality in value only, ignores type
  • === — identity: value AND type must match

⛔ == traps0 == false → TRUE · "" == false → TRUE · "0" == false → TRUE · 1 == true → TRUE · "5" == 5 → TRUE. Always use === for strict checks. 

var_dump(0 == false);   // bool(true) — TRAP!
var_dump(0 === false);  // bool(false) — correct
var_dump("5" === 5);    // bool(false) — type differs

Arrays & Loops

Indexed arrays, associative arrays, and all loop types

Indexed Arrays
  • Elements accessed by numeric index starting at 0
  • Add element: $arr[] = 'value';
  • Remove: unset($arr[2]); then array_values($arr) to reindex
$colors = array('red','blue','green','black');
$colors[] = 'purple';     // add
unset($colors[2]);        // remove 'green'
$colors = array_values($colors); // reindex
Associative Arrays
  • Key-value pairs — keys must be unique
  • Assigning existing key overwrites the value
$favColors = array(
  'Joe'   => 'blue',
  'Elena' => 'green'
);
$favColors['Bob'] = 'purple';  // add 
unset($favColors['Joe']);       // remove
$favColors['Joe'] = 'yellow';  // overwrites!
All Loop Types
Loop Use when Key syntax
while Condition checked before each iteration Check first, may skip entirely
do…while Must run at least once Check after; semicolon after )
for Known number of iterations for(init; cond; step)
foreach Iterating arrays foreach($arr as $val) 
// while
while ($x < 1000) { echo $x; $x = $x * $x; }

// do-while (runs at least once)
do { echo $x; $x = $x * $x; } while ($x < 1000);

// for
for ($i = 1; $i <= 10; $i++) { echo $i . ":" . ($i*$i); }

// foreach — indexed
foreach ($colors as $color) { echo $color; }

// foreach — with index
foreach ($colors as $i => $color) {
  echo "$i => $color";
}

⚠ do…whileDon't forget the semicolon after the closing parenthesis: } while (condition);

Functions & Form Handling

User functions, built-in utilities, input sanitization & validation

Function Syntax
function multiply($x, $y) {
  return $x * $y;
}
// Call
echo multiply(2, 3);       // 6
$a = multiply(2, 3);       // assign result
$b = multiply(multiply(2,3), multiply(3,4)); // 72

// Note: no return = no output even if called
multiply(2, 3);  // prints NOTHING without echo
isset() & empty()
Function Returns TRUE when
isset($v) Variable is declared AND not NULL
empty($v) Variable is not declared, false, null, 0, "", or "0" 
$u = "BroCode";
if(isset($u))  echo "set";       // set
if(!empty($u)) echo "not empty"; // not empty

$u = null;
if(!isset($u)) echo "not set";   // not set
if(empty($u))  echo "empty";     // empty
Sanitize & Validate User Input

⛔ XSS RiskNever echo $_POST['input'] directly — users can inject <script>alert("hacked")</script>. Always sanitize first!

Sanitize Filters

Filter Strips
SANITIZE_SPECIAL_CHARS HTML tags & special chars
SANITIZE_EMAIL Illegal email chars
SANITIZE_URL Illegal URL chars
SANITIZE_NUMBER_INT Non-digit chars (keeps +/−)
SANITIZE_NUMBER_FLOAT Non-numeric (keeps . e E)

Validate Filters

Filter Checks
VALIDATE_INT Valid integer
VALIDATE_FLOAT Valid float
VALIDATE_EMAIL Valid email format
VALIDATE_URL Valid URL
VALIDATE_IP Valid IP address 
// Sanitize then validate pattern:
$email = filter_input(INPUT_POST, "email", FILTER_SANITIZE_EMAIL);
$age   = filter_input(INPUT_POST, "age",   FILTER_VALIDATE_INT);

if(empty($age))   echo "Invalid number";
if(empty($email)) echo "Invalid email";
