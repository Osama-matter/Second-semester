Case Study — Practice Exam 1:
EduTech Academy, ETA, is an Egyptian company that operates private language-learning centres across 18 governorates. Each centre contains four departments: registration, scheduling, finance, and student services. The ETA Head Quarter (HQ) is in Cairo. ETA is facing difficulties tracking student enrolments and payments manually. Management has decided to automate the Registration Department and establish an Information System Centre (ISC) at HQ to provide technical support to all branches. Each branch will be equipped with one computer and one printer connected to servers in ISC via a network. The automated system should allow staff to register students from any branch.

Registration Department roles: Receive student application, Validate student application, Forward validated application to the finance department.

Automated Registration System (ARS) functionalities: Register student application, Validate application, Print application receipt, Send validated application to Finance Department.
Data attributes: Application No., Application Date, Student Name, Student Address, Chosen Courses (Course name, Level, up to 4 courses per application).

Q1 — Fundamentals of System Analysis and Design

9 Marks▼
Part A

Define the term "Information System (IS)".

Model Answer

An Information System (IS) is a set of interrelated components that collect, process, store, and distribute data and information to support decision making, coordination, and control in an organisation. It combines technology, people, and processes to serve organisational goals.

Part B

State two types of Information Systems (IS) and explain one of them in no more than three lines.

Model Answer
  • TPS (Transaction Processing System) – records and processes day-to-day business transactions (e.g., orders, payments). It ensures fast, accurate recording of high-volume routine activities.
  • MIS (Management Information System) – provides regular structured reports to middle management for operational decisions.
  • DSS (Decision Support System) – supports complex, non-routine decisions.
  • ES (Expert System) – captures expert knowledge to assist non-experts.
Part C

State two roles of the System Analyst and explain one in no more than three lines.

Model Answer
  • Systems Analyst – analyses current system problems and proposes solutions by gathering requirements and modelling the system using tools like DFDs.
  • Requirements Analyst – focuses specifically on eliciting and documenting what the system must do.
  • Project Manager – plans, monitors, and controls project scope, time, and cost.

Q2 — Analysis/Design Approaches & Information Requirements

12 Marks▼
Part A

Define System Development Methodology and state its relationship to SDLC.

Model Answer

A System Development Methodology is the physical implementation of the logical lifecycle (SDLC). While SDLC defines the logical phases (what to do), a methodology prescribes how to execute those phases in practice (procedures, tools, rules). Examples include Waterfall, Agile, Spiral, and Prototyping models.

Part B

State three SDLC phases.

Model Answer
  • Planning
  • Analysis
  • Design and Prototyping
  • Implementation (SW Development, Testing, Integration)
  • Configuration
  • Deployment, Operations, and Maintenance
Part C

Define Business Requirements and state two types.

Model Answer

A system requirement is a statement of what the system must do or what characteristics it needs to have. Business requirements can be classified as:

  • Functional Requirements – what the system must do (e.g., register a student).
  • Nonfunctional Requirements – performance, security, availability constraints.
  • User Requirements – usability and user interface requirements.
Part D

Mention two Analysis Strategies and explain one in no more than three lines.

Model Answer
  • Root Cause Analysis – digs beneath surface symptoms to identify the fundamental reason for a problem, preventing recurrence by addressing the true cause rather than its effects.
  • Problem Analysis – evaluates the current system to identify shortcomings and improvement opportunities.
  • Technology Analysis – identifies how emerging technologies may solve existing business problems.

Q7 — Data Storage & Program Design

6 Marks▼
Part A

State two disadvantages of the Flat File storage approach and explain one in no more than three lines.

Model Answer
  • Data Duplication – the same data item may be stored in multiple files, causing redundancy, storage waste, and inconsistency when updates are not applied everywhere.
  • Data Dependence – programs are tightly coupled to file structures; any change in file layout requires modifying all programs that use it.
  • Concurrency Problems – flat files do not support simultaneous access by multiple users efficiently.
Part B

In Modern Structured Program Design, explain the Bottom-up strategy.

Model Answer

In the Bottom-up strategy, program design starts from the lowest-level, most detailed modules and builds upward toward higher-level modules. Individual sub-components are designed and tested in isolation first, then integrated progressively. This approach is useful when reusable low-level components already exist, but it can be harder to maintain overall system coherence compared to top-down.

Q7 — Data Storage & Program Design

6 Marks▼
Part A

State two advantages of the Database approach over flat files and explain one.

Model Answer
  • Controlling Redundancy – a DBMS stores each data item in one place, eliminating duplicate copies that cause inconsistency and wasted storage.
  • Data can be shared – multiple users and applications can access the same data concurrently.
  • Restricting unauthorised access – DBMSs enforce security at field and record level.
Part B

In Modern Structured Program Design, explain the Top-down strategy.

Model Answer

In the Top-down strategy, the main program module is designed first, then progressively decomposed into sub-modules of increasing detail. The design begins with a high-level overview (e.g., the main dispensing process) and breaks it down into smaller, manageable components (e.g., validate prescription, print label). This ensures the overall system logic is defined before detail, improving clarity and maintainability.

Q7 — Data Storage & Program Design

6 Marks▼
Part A

State two disadvantages of the Database approach and explain one in ≤ 3 lines.

Model Answer
  • Complexity – a DBMS is a sophisticated software system requiring skilled administrators and developers to design, maintain, and optimise, adding organisational overhead.
  • Cost of DBMS – commercial DBMS licences can be expensive, especially for large-scale deployments.
  • Higher impact of a failure – a DBMS failure can affect all applications that depend on it simultaneously.
Part B

In Modern Structured Design, explain the Top-down strategy.

Model Answer

Design starts at the highest-level main module and is progressively decomposed into lower-level sub-modules until all detailed functions are defined. This approach ensures the overall program structure and logic are clear before detail is added, improving readability and maintainability.

Q7 — Data Storage & Program Design

6 Marks▼
Part A

State two advantages of the Flat File approach and explain one in ≤ 3 lines.

Model Answer
  • Simple organisation and accessing methods – flat files use straightforward sequential or indexed access methods that are easy to implement and understand, without requiring complex query languages or schema design.
  • Cheap – no DBMS software licence costs.
Part B

State the three Program Design Architectures and briefly describe each.

Model Answer
  • Traditional Monolith – single deployment unit, shared database, centralised codebase. Simple but hard to scale independently.
  • Microservices – multiple independent services, each with its own database and deployment. Highly scalable but complex.
  • Modular Monolith – modular design within a single deployment unit, providing maintainability benefits without distributed system complexity.
