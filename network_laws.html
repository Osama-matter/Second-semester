LEC 03 Data & Signals — Laws
Bits per Signal Level
How many bits can one signal level carry
n = log₂ L
▾
n = log₂ L
Variables
nNumber of bits per level
LNumber of signal levels
Key Rules
⚠Result must be a whole number (power of 2). If not — round UP to next power of 2
📌 8 levels → log₂ 8 = 3 bits. 9 levels → log₂ 9 ≈ 3.17 → round up → 4 bits.
Nyquist Theorem — Noiseless Channel
Maximum bit rate of a noiseless channel
BR = 2 · B · log₂ L
▾
BitRate = 2 × B × log₂ L
Variables
BRBit Rate (bps)
BBandwidth of channel (Hz)
LNumber of signal levels
Rearranged — Find L
log₂ L = BR / (2 × B)
L = 2^(BR / 2B)
Examples
B=3000 Hz, L=2 → BR = 2×3000×1 = 6,000 bps
B=3000 Hz, L=4 → BR = 2×3000×2 = 12,000 bps
B=20 kHz, BR=265 kbps → L = 2^6.625 ≈ 98.7 → use L=64 or L=128
📌 Nyquist gives a THEORETICAL maximum for a NOISELESS channel. Real channels have noise, so use Shannon too.
Shannon Capacity — Noisy Channel
Absolute maximum rate regardless of signal levels
C = B · log₂(1+SNR)
▾
C = B × log₂(1 + SNR)
Variables
CChannel Capacity (bps) — the CEILING
BBandwidth (Hz)
SNRSignal-to-Noise Ratio (linear, not dB)
SNR Conversion from dB
SNR_dB = 10·log₁₀(SNR)
SNR = 10^(SNR_dB / 10)
Special Cases
SNR = 0 → C = B × log₂(1) = 0 bps (pure noise = no data)
SNR = 3162, B = 3000 → C = 3000 × 11.62 ≈ 34.86 kbps
SNR_dB = 36, B = 2 MHz → SNR ≈ 3981 → C ≈ 23.9 Mbps
📌 Shannon = CEILING. Nyquist = HOW MANY LEVELS to reach your chosen rate. Use both together: Shannon first, Nyquist second.

Propagation & Transmission Time
Two types of delay in a network link
Tp = d/v · Tt = s/BW
▾
Propagation Time
Tp = Distance / Speed
TpTime for a signal to travel the medium
dDistance (meters)
vPropagation speed (m/s)
Transmission Time
Tt = Message Size / Bandwidth
TtTime to push all bits onto the wire
SizeMessage size (bits)
BWBandwidth (bps)
When Each Dominates
Short msg + high BW → Propagation dominates (2.5 KB @ 1 Gbps: Tt=0.02ms, Tp=50ms)
Large msg + low BW → Transmission dominates (5 MB @ 1 Mbps: Tt=41.94s, Tp=50ms)
Throughput
Actual data transfer rate (always ≤ bandwidth)
T = (frames × bits) / time
▾
Throughput = (Frames/sec × Bits/frame)
Example
12,000 frames/min × 10,000 bits/frame ÷ 60 s = 2,000,000 bps = 2 Mbps
(even though bandwidth = 10 Mbps — overhead reduces actual throughput)
📌 Throughput ≤ Bandwidth always. Network overhead (headers, retransmissions) reduces throughput below the stated bandwidth.
LEC 04 Multiplexing — FDM & TDM Laws
FDM — Frequency Division Multiplexing
Minimum bandwidth with guard bands
BW = n·ch + (n−1)·g
▾
Min BW = (n × ch_BW) + ((n−1) × guard)
Variables
nNumber of channels
ch_BWBandwidth of each channel (Hz)
guardGuard band width (Hz)
n−1Number of guard bands (between channels)
Example
5 channels × 100 kHz
Guard band = 10 kHz

Min BW = (5×100) + (4×10)
= 500 + 40 = 540 kHz
📌 Guard bands = channels − 1. They prevent interference between adjacent channels.
AMPS — Channel Count Formula
Number of users from total bandwidth
channels = BW / ch_BW
▾
Formula
Total channels = Total BW / Per-user BW
Voice channels = Total − Control channels − rounding
AMPS Example
Band 824–849 MHz → 25 MHz total ÷ 30 kHz per user = 833 channels
− 42 control channels − 1 rounding = 790 voice channels
TDM — Time Division Multiplexing Laws
Slot, frame, and rate calculations
out_slot = in_slot / n
▾
Core Formulas
Output slot = Input slot / n
Link rate = n × input rate
Frame rate = any input rate
Frame size = n × slot size
Variables
nNumber of input connections
slotDuration of one time slot
frameOne cycle containing n slots
Examples
3 × 1-kbps: Input slot = 1ms → Output slot = 1ms/3 ≈ 0.333ms → Frame = 1ms
4 × 1-Mbps: Input dur = 1μs → Output dur = 0.25μs → Rate = 4 Mbps → Frames = 1M/s
4 × 1-kbps: Rate = 4 kbps → Slot = 250 μs → Frame = 1 ms
📌 Golden rule: Frame duration = Input bit duration (they are always equal). This is how you verify your calculation.
LEC 06 Error Detection — Checksum Laws
Basic Checksum Rule
Detect errors using sum of data values
Σdata + checksum = 0
▾
Method A — Send Positive Sum
Sender sends: (data, sum)
Receiver: adds data → must equal sum
If equal → NO ERROR ✓
Method B — Send Negative Sum (Preferred)
Sender sends: (data, −sum)
Receiver: adds ALL values
Result = 0 → NO ERROR ✓
Result ≠ 0 → ERROR ✗
📌 Method B is always preferred: receiver just checks if sum of ALL values (data + checksum) = 0.
One's Complement Arithmetic
The math behind the Internet checksum
invert all bits
▾
Represent Positive Number
Convert to binary
If more bits than word size:
Add carry bit back (wrap-around)

21 → 10101 (5 bits)
Split: 0101 + 1 = 0110 = 6
Represent Negative Number
Take positive representation
INVERT ALL BITS (0↔1)

+6 = 0110
−6 = 1001 (= 9 unsigned)
Internet Checksum Process (Sender → Receiver)
1Convert data to binary; group into 16-bit words
2Add all words using one's complement addition (wrap any carry)
3Complement the sum (invert all bits) → this is the CHECKSUM
4Receiver adds data + checksum → result must = all 1s (FFFF) → complement → 0000 = NO ERROR
LEC 07 Ethernet — Address & Frame Laws
MAC Address Type — Identification Rule
Unicast / Multicast / Broadcast
LSB of byte 1
▾
Check LSB (bit 0) of the FIRST byte
Decision Rule
LSB = 0 (even hex digit) → UNICAST
LSB = 1 (odd hex digit) → MULTICAST
All bytes = FF → BROADCAST
Examples
4A:... → 'A'=1010 → LSB=0 → UNICAST
47:... → '7'=0111 → LSB=1 → MULTICAST
FF:FF:FF:FF:FF:FF → BROADCAST
📌 Only look at the LAST binary digit of the FIRST byte's last hex digit. Even = Unicast, Odd = Multicast.
Ethernet Bit Transmission Order
How MAC address bits are sent on the wire
LSB first per byte
▾
The Two Rules
Rule 1: Bytes are sent LEFT → RIGHT (normal order)
Rule 2: Within each byte, bits are sent RIGHT → LEFT (LSB first = reversed)
Example: First byte = 47 = 0100 0111
Written: 0 1 0 0 0 1 1 1
Sent: 1 1 1 0 0 0 1 0 ← bits reversed (right to left)
Ethernet 802.3 Frame Size
Minimum and maximum frame calculation
64 – 1518 bytes
▾
Frame Fields (Overhead)
Preamble: 7 bytes
SFD: 1 byte
Dest MAC: 6 bytes
Source MAC: 6 bytes
Length/Type: 2 bytes
CRC: 4 bytes
─────────────────────
Total overhead: 26 bytes
Data Field + Total
Data: 46 (min) – 1500 (max) bytes

Min total = 26 + 46 = 72 bytes
→ without preamble+SFD = 64 bytes

Max total = 26 + 1500 = 1526 bytes
→ without preamble+SFD = 1518 bytes
📌 Standard sizes quoted exclude preamble & SFD: Min = 64 bytes, Max = 1518 bytes.
