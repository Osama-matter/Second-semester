Foundations & Selectors

CSS (Cascading Style Sheets) controls presentation of HTML content. A styling language — NOT a programming language. Mastering selectors is the first step to CSS expertise.

Styling Language Selector-Based
📋 Key Concepts
  • CSS = Cascading Style Sheets — for presentation of HTML content
  • Cascading = styles on parent elements inherit to all children
  • Rule = Selector + Declaration (property: value;)
  • 3 CSS Types: Inline → Internal → External
  • Specificity Order: Inline overrides Internal overrides External overrides Browser
  • 4 Selector Types: Element, Class (.), ID (#), Universal (*)
🔑 Selector Combinations
  • Multi-selector (no space): #id.class — element must have BOTH
  • Ancestor-child (space): .parent .child — nested targeting
  • Group (comma): .a, .b {} — share same rules
  • Universal (*): targets every element — use for defaults only
  • Class: reusable, element can have multiple classes
  • ID: unique per page, element can only have ONE id
⚠ Critical Distinction
An element can have multiple classes but only ONE ID. IDs must be unique across the entire webpage. Failing to respect this breaks both CSS specificity and JavaScript targeting.
🚫 Common Mistake
CSS is NOT a programming language — it has no variables (vanilla), no loops, no conditionals. It is purely declarative. Confusing this in exam answers costs marks.
CSS Syntax & Types Reference
CSS Syntax 
/* Selector { property: value; } */

h1 {
  color: blue;
  font-size: 24px;
}

/* Class Selector */
.btn { background: green; }

/* ID Selector */
#header { color: red; }

/* Universal Selector */
* { font-family: Arial; }
3 Ways to Apply CSS 
/* 1. INLINE (highest priority) */
<h1 style="color:blue;">Title</h1>

/* 2. INTERNAL (in <head>) */
<style>
  h1 { color: blue; }
</style>

/* 3. EXTERNAL (separate .css file) */
<link rel="stylesheet" 
      href="style.css">
CSS Type Location Scope Priority Best For
Inline Inside HTML tag style="" Single element HIGHEST Quick one-off fixes
Internal <style> in <head> One page Medium Single-page styles
External Separate .css file Entire site Lowest ⭐ Recommended always
✓ Advantages of CSS
Saves Time (write once, apply everywhere) · Faster Page Loads · Easy Maintenance (change one file, update whole site) · Mobile Device Compatibility · Superior Styles vs HTML attributes

Box Model & Layout Properties

Every HTML element is a box. Mastering the box model — margins, padding, borders, and sizing — is fundamental to all CSS layout work.

Box Model Text Formatting Background
📋 Key Box Model Concepts
  • Content: the actual text/image inside the element
  • Padding: space BETWEEN content and border (inside)
  • Border: surrounds padding and content
  • Margin: space OUTSIDE the border (between elements)
  • border-radius: rounds the corners of the border box
  • border-collapse: collapse | separate — for tables
🎨 Text & Font Properties
  • font-weight: lighter | normal | bold | bolder | 100–900
  • font-style: normal | italic | oblique
  • font-variant: normal | small-caps
  • text-indent: indents first line (negative values push left)
  • letter-spacing: space between characters
  • line-height: space between lines (no negative values)
  • word-spacing: space between words (negative allowed)
  • white-space: nowrap | pre | pre-line | pre-wrap
⚠ Padding vs Margin — Know the Difference!
Padding = space INSIDE the border (affects element's background area). Margin = space OUTSIDE the border (transparent, between elements). Both can be set individually: padding-top, margin-left, etc.
Box Model Code Reference
Border Properties 
p {
  border: 2px solid powderblue;
  border-radius: 10px;
  border-top-style: dotted;
  border-bottom-style: dashed;
  border-left-style: double;
  border-left-width: 10px;
  border-left-color: silver;
}
Padding & Margin 
div {
  padding: 30px;          /* all sides */
  padding-top: 10px;    /* top only */
  margin: 50px;           /* all sides */
  margin-left: 20px;    /* left only */
  
  /* Shorthand: top right bottom left */
  margin: 10px 20px 10px 20px;
}
Background Properties
Background Styling 
body {
  background-image: url("myBackground.jpg");
  background-repeat: no-repeat;        /* no-repeat | repeat | repeat-x | repeat-y */
  background-attachment: fixed;          /* fixed | scroll */
  background-position: center;
  background-size: cover;               /* cover | contain | auto */
  
  /* OR use gradient */
  background: linear-gradient(skyblue, lightgreen);
  background: linear-gradient(to right, skyblue, lightgreen);
}
Property Key Values Notes
font-weight lighter | normal | bold | bolder | 100-900 Not all browsers support numeric
line-height normal | number | length | initial ❌ Negative values NOT allowed
word-spacing normal | length | initial ✅ Negative values allowed
text-indent px | em | % (positive or negative) Negative = indent left (outdent)
white-space normal | nowrap | pre | pre-line | pre-wrap nowrap prevents text wrapping
float left | right | none Use clear: both to stop float
💡 Float Property
float positions an element left or right of its container. Popular for wrapping text around images. Use clear: none | left | right | both | initial | inherit on subsequent elements to prevent wrapping interference.

Positioning, Pseudo-Elements & Navigation

CSS positioning controls exact element placement. Pseudo-elements style specific parts of elements. Together with navigation bars, these form the backbone of web page structure.

CSS Position Pseudo-Elements NavBar
📍 CSS Position Values
  • static — default, normal document flow, top/left have no effect
  • relative — offset from its OWN normal position
  • absolute — positioned relative to nearest POSITIONED ancestor
  • fixed — positioned relative to the VIEWPORT (stays on scroll)
  • sticky — switches between relative and fixed based on scroll
✨ Pseudo-Elements Reference
  • ::first-line — styles first line of block-level text
  • ::first-letter — styles first character of block-level text
  • ::before — inserts content BEFORE element content
  • ::after — inserts content AFTER element content
  • ::marker — styles list item markers/bullets
  • ::selection — styles user-selected/highlighted text
  • ::placeholder — styles input placeholder text
⚠ Position: Absolute — Key Rule
position: absolute positions an element relative to its nearest positioned ancestor (any ancestor with position ≠ static). If no such ancestor exists, it positions relative to the <html> element. The parent MUST have a position set!
Positioning Code Examples
Position Property 
/* Relative - offset from self */
#box1 {
  position: relative;
  left: 100px;
  top: 50px;
}

/* Absolute - relative to parent */
#box2 {
  position: absolute;
  top: 100px;
  left: 100px;
}
Pseudo-Elements 
/* Syntax: selector::pseudo-element {} */

p::first-line {
  color: #ff0000;
  font-variant: small-caps;
}

p::first-letter {
  font-size: xx-large;
  color: red;
}

p::before { content: '★ '; }
p::after  { content: ' ★'; }
Navigation Bar Patterns
Horizontal NavBar (Floating Method) 
ul { list-style-type: none; margin: 0; padding: 0; 
     overflow: hidden; background-color: #333; }

li { float: left; }            /* horizontal layout */

li a { display: block; color: white; padding: 14px 16px;
        text-decoration: none; text-align: center; }

li a:hover:not(.active) { background-color: #111; }
.active { background-color: #04AA6D; }

/* Right-align a link */
<li style="float:right"><a href="#about">About</a></li>
NavBar Type Key CSS Used Notable Feature
Vertical display: block on li a Each link fills full width
Horizontal (inline) display: inline on li No height/width control
Horizontal (float) float: left on li ⭐ Allows padding/height/width
Fixed Top/Bottom position: fixed; top: 0 Stays visible on scroll
Full-height Sidebar position: fixed; height: 100% Use overflow: auto
💡 CSS Pseudo-Classes for Links
a:link (unvisited) · a:visited · a:hover (mouse over) · a:active (being clicked). Use :not(.active) to apply hover only to non-active links. Always keep this order: LVHA!

Transforms & Animation

CSS Transforms let you move, rotate, scale, and skew elements. CSS Animations let elements gradually change styles over time using keyframes — no JavaScript required.

Transform @keyframes Animation
🔄 Transform Functions
  • translate(x,y) — moves element from current position
  • rotate(deg) — rotates clockwise (+) or counter-clockwise (-)
  • scale(n) — resizes element (0.5 = 50%, -1 = flip)
  • scaleX(n) / scaleY(n) — scale one axis only
  • skew(xdeg, ydeg) — skews on X and Y axes
  • skewX(deg) / skewY(deg) — skew one axis only
  • matrix() — combines all transforms in one function
⏱ Animation Properties
  • animation-name — references @keyframes name
  • animation-duration — time in seconds (default: 0s)
  • animation-delay — delay before start (negative = starts mid)
  • animation-iteration-count — number | infinite
  • animation-direction — normal | reverse | alternate
  • animation-timing-function — ease | linear | ease-in | ease-out
  • animation-fill-mode — none | forwards | backwards | both
🚫 Animation Default Behavior
By default, when an animation finishes it returns to its original style. To keep the final state, set animation-fill-mode: forwards. Default duration is 0s — always set animation-duration or nothing will play!
Transform Code Examples
CSS Transforms 
/* translate: move 50px right, 20px down */
div { transform: translate(50px, 20px); }

/* rotate: 20 degrees clockwise */
div { transform: rotate(20deg); }

/* scale: half size | flip | one axis */
div { transform: scale(0.5); }
div { transform: scale(-1); }  /* flip! */
div { transform: scaleX(0.5); }

/* skew: tilt along X and Y */
div { transform: skew(50deg, 20deg); }

/* Modern shorthand properties */
div { translate: 25px 25px; }
div { rotate: 20deg; }
div { scale: 0.5; }
CSS Animation 
div {
  animation-name: example;
  animation-duration: 4s;
  animation-delay: 2s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
  animation-fill-mode: forwards;
}

@keyframes example {
  0%   { background-color: red;   }
  25%  { background-color: yellow; }
  50%  { background-color: blue;  }
  100% { background-color: green; }
}
Property Values Notes
animation-direction normal | reverse | alternate | alternate-reverse alternate = forward then backward
animation-timing-function ease | linear | ease-in | ease-out | ease-in-out ease is default (slow-fast-slow)
animation-fill-mode none | forwards | backwards | both forwards = keep last frame style
animation-iteration-count number | infinite infinite loops forever
animation-delay time in seconds Negative value = animation already started
✓ Pro Tip: from/to vs Percentages
Use from {} to {} for simple 2-state animations. Use 0% {} 25% {} 50% {} 100% {} for multi-step animations. You can animate multiple CSS properties simultaneously within the same keyframe step.

CSS Flexbox Layout

Flexbox (Flexible Box Layout) provides an efficient way to lay out, align and distribute space among items in a container. The container alters items' width/height to fill available space.

Container Props Item Props Alignment
🔑 The Golden Rule: Set display: flex on the container (parent). Child elements automatically become flex items. All flexbox properties then apply to control their layout behavior.
📋 Container Properties
  • display: flex — activates flexbox on the container
  • flex-direction — row | row-reverse | column | column-reverse
  • flex-wrap — nowrap | wrap | wrap-reverse
  • flex-flow — shorthand for direction + wrap
  • justify-content — aligns items on MAIN AXIS (horizontal)
  • align-items — aligns items on CROSS AXIS (vertical)
  • align-content — aligns multiple FLEX LINES
  • gap / row-gap / column-gap — space between items
📦 display Values Reminder
  • block — takes full width, starts on new line
  • inline — no width/height control, flows in text
  • inline-block — inline but supports width/height
  • flex — ⭐ activates flexbox container
  • none — hides element completely (removed from flow)
FLEX AXES
📐 Main Axis = set by flex-direction (default: horizontal)
📐 Cross Axis = perpendicular to main axis
⚠ align-items vs align-content
align-items aligns individual flex items on the cross axis within a single line. align-content aligns the flex LINES themselves — it has no effect when there is only one line (needs flex-wrap: wrap to have multiple lines).
Flexbox Quick Reference
Property Values Axis Applied To
flex-direction row | row-reverse | column | column-reverse Sets main axis Container
flex-wrap nowrap | wrap | wrap-reverse — Container
justify-content flex-start | flex-end | center | space-between | space-around | space-evenly Main axis Container
align-items stretch | flex-start | flex-end | center | baseline | normal Cross axis Container
align-content stretch | center | flex-start | flex-end | space-between | space-around Cross axis Container (multi-line)
gap length | row-gap column-gap Both Container
Flexbox Code
Container Setup 
.flex-container {
  display: flex;
  flex-direction: row;         /* or column */
  flex-wrap: wrap;
  justify-content: center;
  align-items: stretch;
  gap: 10px;
  background-color: DodgerBlue;
}

/* Shorthand: flex-flow = direction + wrap */
.flex-container {
  flex-flow: row wrap;
}
justify-content Values 
/* center: items at center */
justify-content: center;

/* flex-start: items at start (default) */
justify-content: flex-start;

/* flex-end: items at end */
justify-content: flex-end;

/* space-between: equal space between */
justify-content: space-between;

/* space-around: equal space around */
justify-content: space-around;

/* space-evenly: equal space all around */
justify-content: space-evenly;
✓ Perfect Centering with Flexbox
To center an element both horizontally and vertically: set the container to display: flex; justify-content: center; align-items: center; — This is the modern, clean solution that replaced the old table/position hacks.
💡 Exam Quick Summary
Flexbox works with a parent container and its child items. Container properties control the overall layout direction and alignment. The overflow: scroll/auto/hidden property also works alongside flexbox when content exceeds container dimensions.
