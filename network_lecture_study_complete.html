🔄 Client–Server vs Peer-to-Peer

Client–Server: Client (requesting machine) sends a request → Server (service machine) sends back a reply. The server provides the service and the client consumes it.

Peer-to-Peer (P2P): Each device is both a client and a server at the same time — no fixed roles. Devices communicate directly without a dedicated server. Example: any machine can be either a Client or a Server.

📡 Data Communication

Definition

Data Communication = exchange of data between two devices via a transmission medium. Data = information presented in any form (text, audio, video). Requires two devices that are synchronized and an agreed-upon protocol between sender and receiver.

📍

Local Communication

Communication over a short distance (e.g. same room or building). Uses Point-to-Point connection.

Short Distance
🌍

Remote / Telecommunication

Communication over long distances. Uses telephone, telegraph, television infrastructure.

Long Distance

✅ Characteristics of Data Communication

📬

Delivery

Data must be delivered to the correct destination — the right person and the right device only.

🎯

Accuracy

Data must be delivered without any change during transmission — no corruption or alteration of bits.

⏱️

Timeliness

Data must be delivered in time — at the right moment. The entire message should arrive on time.

📦

Jitter

Variation in packet arrival time. For no jitter: Packet1 arrival time = Packet2 arrival time. Uneven arrival → distortion (e.g. choppy audio/video).

⚡ Timeliness vs Jitter — the difference

Timeliness: the entire message arrives on time.
Jitter: measures when a sound/video file arrives — specifically the variation in arrival time between packets. If Packet1 arrives at t=1 but Packet2 arrives at t=3 (instead of t=2), there is jitter → this causes choppy sounds and cuts.

⚙️ Data Communication System Components

M

Message

The data to be transferred — can be text, number, or audio.

S

Sender

Device that sends data — a computer. It transmits the message.

R

Receiver

Device that receives the message — a computer. Takes incoming data.

Me

Medium

Physical path for data transfer from sender to receiver. Examples: twisted pair, coaxial cable.

P

Protocol

Rules that govern data communication — an agreement between two devices on how to communicate. Without protocol, two devices may be connected but cannot communicate.

➡️

Example: Keyboard → Computer

🌐 Network Definition & Criteria

What is a Network?

Network = a set of devices connected by communication links.
Node = any device able to send/receive data (e.g. computer, printer).
Link = communication pathway that transfers data from one device to another.
Distributed Processing: a task is divided between multiple computers instead of one large computer.

⚡

Performance

Measured by transit time (time message takes to travel from device to device) and response time (time between inquiry and response). Factors: number of users, transmission medium, hardware, software.

🛡️

Reliability

Measured by: (1) Frequency of failure, (2) Recovery time after failure, (3) Robustness in catastrophe. Protected by backup network.

🔒

Security

Protects data from: unauthorized access, damage, development. Also covers data recovery (breach/loss) and disaster recovery plans.

🗺️ Physical Topology

Definition

Physical Topology = the way a network is laid out physically — the geometric representation of links and nodes (shape + connection). At least 2 devices must connect together to form a topology.

🔵

Point-to-Point Link

Dedicated link between exactly 2 devices. The entire capacity of the link is reserved for those 2 devices only. Uses wire cable, microwave, or satellite.

Dedicated
🔶

Multipoint Link (Multidrop)

More than 2 devices share the same link. Capacity is shared — either spatially (devices use link at same time) or temporally (users take turns).

Shared
🕸️

① Mesh Topology

Every link is dedicated point-to-point. Links carry data only between 2 devices (Full Duplex).

Formula: Links = n(n-1)/2  |  Ports per device = n-1

✅ No traffic problem (each link has own data)
✅ Robust — failure doesn't affect others
✅ Privacy & Security
✅ Easy fault identification
❌ Large amount of cables & ports
❌ Difficult installation & expensive
Used as backbone in hybrid networks
⭐

② Star Topology

Every link is dedicated point-to-point to a central controller (Hub). No direct traffic between devices — Hub acts as exchange.

✅ Less cabling → less expensive
✅ Easy installation & fault identification
✅ Each device has 1 link & 1 port
✅ Robust — one failure doesn't affect others
❌ Dependency on one single point (Hub)
❌ If Hub breaks → entire network down
Used in LAN
🚌

③ Bus Topology

Multipoint — one long cable acts as backbone. Nodes connect via drop lines and taps. Signal travels the backbone, loses energy → heat → weak signal → limits nodes and distance.

✅ Easy to install
✅ Less cables → less expensive
❌ Difficult fault identification
❌ Hard to add more devices
❌ If backbone breaks → transmission stops
❌ Signal reflects at end → returns → causes noise
💍

④ Ring Topology

Every device connects to exactly 2 neighbors only. Signal moves in one direction. Each device contains a repeater to regenerate signals.

✅ Easy installation & fault identification
❌ Unidirectional — one device failure stops the cycle
Dual Ring: has a primary + secondary ring. Can switch cable to close a break → protects transmission.

🔀 Hybrid Topology

A combination of more than one topology at the same time. Exists because multiple topologies are needed simultaneously in large networks.

📏 Network Types by Size

🏠

LAN — Local Area Network

Covers area less than 2 miles. Privately owned. Devices are close together (same building/home). Allows sharing of resources (HW & SW) and data. High data rate (4→16 Mbps to 100→1000 Mbps).

Distinguished by: Size, Transmission Medium (one type), Topology (star, bus, mesh)

Private · High Speed

Covers large geographic area and long distances (country, world). Technologies: Circuit Switching, Packet Switching, Frame Relay, ATM.

Packet Switching: Data split into packets, sent independently, arrive out of sequence
Large · Long Distance
🏙️

MAN — Metropolitan Area Network

Size between LAN and WAN. Inside a town/city. Larger geographic area than LAN. Crosses public ways — uses Common Carrier Circuits (high speed DSL).

City-Wide
🔀

Switched WAN

Data represented as information transferred through nodes. Uses ATM and Frame Relay. Forms the backbone of the Internet.

ATM / Frame Relay
📞

Point-to-Point WAN

Connects two devices using a line — either dual line (temporary) or leased line (permanent). Uses ISDN: converts analog connections to digital and stays on at the same time.

Narrow Band: 64 Kbps · Circuit Switching Broad Band: 100 Mbps · Packet Switching

📜 Protocols & Standards

Protocol

A set of rules that govern data communication — defines What, How, and When it is communicated. It is a standard (agreed-upon rules). Three elements:

📐

Syntax

Structure and format of data — how the bits are organized and arranged.

💬

Semantics

Meaning of the bits — what each section of bits means and what action should follow.

⏰

Timing

When data is sent and how fast it is sent. Must be agreed by sender and receiver to avoid overload and loss.

📈

De Facto Standard

Not officially approved but became standard by common use — people started using it and it spread. Example: internet protocols adopted through widespread use.

🏛️

De Jure Standard

Officially approved — more important. Provides guidelines for manufacturers and open market. Standards bodies: ISO, ITU, IEEE.

🌐 Internet Standard Process

A thoroughly tested specification useful to be adhered to — ensures devices from different manufacturers can communicate. Stages: (1) Internet Draft (valid 6 months) → (2) Published as RFC (Request for Comment) → (3) RFC is edited and numbered.

01 — Foundation

What is a Network?

🔧

Hardware (HW)

Physical equipment — physical devices (اجهزة فيزيائية). Carries signals from device to device through the transmission medium.

Physical Equipment
💾

Software (SW)

Instructions (تعليمات) that make possible the services expected from the network — controls how it all works together.

Instructions & Services

💡 Key Concept

A Network = HW + SW working together to transfer data from one device to another. The communication system uses layers — each assigned a specific task in an ordered hierarchy, each layer has a specific function and works independently.

⭐ Critical Rule

Each layer uses the services of the layer below it. Sender goes from layer 7 → down to 1. Receiver goes from layer 1 → up to 7. Any change in one layer does NOT affect other layers.

    04 — Data & Signals

    Chapter 3: Data and Signals

    Why Signals?

    One major function of the physical layer is to move data in the form of electromagnetic signals across the transmission medium. Data must first be changed to a form the transmission media can accept.

    📊 Analog Signal

    TypeContinuous
    ValuesInfinitely many
    Use casePeriodic analog signals
    BandwidthLess bandwidth

    ⬛ Digital Signal

    TypeDiscrete
    ValuesLimited (often 0 or 1)
    Use caseNon-periodic digital
    RepresentsVariation in data

    Digital Signal Key Concepts

    📐

    Bit Rate

    Number of bits sent in 1 second (bps). More levels = more bits per second.

    bps (bits per second)
    📏

    Bit Length

    Distance one bit occupies on the transmission medium.

    = propagation speed × bit duration
    🔢

    Signal Levels → Bits

    If a signal has L levels, each level needs log₂L bits. Example: 8 levels → log₂8 = 3 bits/level.

    bits/level = log₂L

    Transmission Types

    📡

    Baseband Transmission

    Sending digital signal over a channel without changing it to analog. Requires a low-pass channel — bandwidth starts from zero (single channel medium).

    📌 Key Note

    A digital signal is a composite analog signal with infinite bandwidth.

    📶

    Broadband Transmission (Modulation)

    Changing digital signal to analog for transmission. Uses a band-pass channel — bandwidth doesn't need to start from zero.

    📌 Note

    A low-pass channel can be considered a bandpass channel with lower frequency = 0.

    05 — Transmission Impairment

    What Goes Wrong in Transmission?

    Signals travel through imperfect media — signal at beginning ≠ signal at end. What is sent ≠ what is received.

    📉

    Attenuation

    Loss of energy. Measured in decibels (dB). Signal gets weaker as it travels. Fixed with amplifiers.

    dB = 10 log₁₀(P₂/P₁)
    -3 dB = lose half the power
    🌀

    Distortion

    Signal changes its form or shape. Different frequencies travel at different speeds — components arrive out of phase.

    📡

    Noise

    Corrupts the original signal. Four types:

    Thermal
    Induced
    Crosstalk
    Impulse

    SNR — Signal-to-Noise Ratio

    SNR = average signal power / average noise power. High SNR = signal less corrupted. Low SNR = signal dominated by noise. SNR can also be measured in dB: SNR(dB) = 10 log₁₀(SNR).

    06 — Data Rate Limits

    How Fast Can We Send Data?

    Data rate depends on 3 factors: bandwidth available, level of signals, quality of channel (noise level).

    Nyquist Bit Rate — Noiseless Channel

    BitRate = 2 × B × log₂L

    B = bandwidth (Hz) · L = number of signal levels · BitRate = max bits per second.
    Note: Increasing levels may reduce reliability!

    Shannon Capacity — Noisy Channel

    C = B × log₂(1 + SNR)

    Defines max theoretical capacity regardless of method. Shannon gives upper limit; Nyquist tells you how many levels you need.

    SNR in dB

    SNR(dB) = 10 log₁₀(SNR)

    Convert SNR dB to linear: SNR = 10^(SNR_dB/10). When SNR is very high: C ≈ B × SNR_dB/3.

    Attenuation / Gain in dB

    dB = 10 log₁₀(P₂/P₁)

    Negative dB = loss. Positive dB = gain. dBs can be added when cascading segments. -3 dB = half power. +10 dB = 10× power.

    07 — Network Performance

    How Good is the Network?

    B

    Bandwidth

    Potential speed of the link. Measured in two ways: (1) Hz — range of frequencies a channel can pass. (2) bps — speed of bit transmission. Bandwidth is the maximum possible rate.

    T

    Throughput

    Actual speed — how fast we can actually send data. Always T < B. Bandwidth is potential; throughput is actual.

    Throughput = (frames × bits/frame) / time
    L

    Latency (Delay)

    How long for entire message to arrive at destination from when first bit was sent. Made of 4 components:

    Latency = Propagation + Transmission + Queuing + Processing
    BD

    Bandwidth-Delay Product

    B × D = number of bits that can fill the link. Think of a link as a pipe: cross-section = bandwidth, length = delay, volume = bandwidth × delay. For full-duplex max use: burst size = 2 × B × D.

    J

    Jitter

    Deviation/displacement of signal pulses. Problem when different packets encounter different delays — especially bad for time-sensitive apps (audio, video). Example: packet 1 arrives in 20ms, packet 2 in 45ms, packet 3 in 40ms → jitter!

    Latency Components in Detail

    Propagation Time

    = Distance / Propagation Speed

    Time for a bit to travel from source to destination. Speed of light in cable ≈ 2.4 × 10⁸ m/s. Light in vacuum = 3 × 10⁸ m/s.

    Transmission Time

    = Message Size / Bandwidth

    Time to put the message on the wire. Short message + high BW → propagation dominates. Long message + low BW → transmission dominates.

    08 — Addressing

    4 Levels of Addressing

    1

    Physical Address (MAC Address)

    Lives in Data Link layer. Changes hop-to-hop. 6 bytes = 48 bits = 12 hexadecimal digits. Found in frame header (destination MAC + source MAC).

    AA:BB:CC:DD:EE:FF
    2

    Logical Address (IP Address)

    Lives in Network layer. Stays the SAME end-to-end (because it identifies source and destination). Has two parts: Network ID + Host ID. Example: 172.19 = Network, 0.0 = Host.

    192.168.1.1
    3

    Port Address

    Lives in Transport layer. Identifies the process/application. 16 bits long (represented in port numbers). Remains the same. Used in TCP and UDP.

    :80 :443 :22
    4

    Specific Address

    User-friendly address. Email, URL. Device resolves it internally using logical (IP) + port. Example: google.com resolves to IP + port.

    example.com

    🔑 Physical vs Logical Address

    Physical address CHANGES from hop to hop (MAC is rewritten at each router). Logical address STAYS THE SAME throughout the journey because it identifies source and destination end-to-end.

    Protocol Functions

    📋 Five Protocol Functions

    ✂️

    Segmentation & Reassembly

    Message is divided into transmittable segments, each with a sequence number. The transport layer reassembles at destination and identifies/replaces lost packets.

    Layer 4 — Transport
    📦

    Encapsulation

    Adding control information to data: Address (sender/receiver), Error-detecting code (frame check sequence), and Connection control info.

    Headers & Trailers
    🔌

    Connection Control

    Connectionless: each packet carries full destination address, routed independently (packets may arrive out of order). Connection-oriented: establish → transfer → terminate.

    Connectionless / Oriented
    🔀

    Multiplexing

    Combines multiple signals over a single data link. When bandwidth > device needs, the link can be shared. Saves cost — fewer circuits needed.

    FDM / TDM / STDM
    ⚙️

    Transmission Services

    Priority, service quality, and security controls offered by the protocol to upper layers for reliable data delivery.

    QoS
    📶

    Bandwidth Utilization

    Wise use of available bandwidth. Efficiency → multiplexing. Privacy & anti-jamming → spreading. Goal: achieve specific communication objectives.

    Key Concept

    ✂️ Segmentation vs Reassembly

    📤

    Segmentation (at Sender)

    A message is divided into smaller transmittable segments. Each segment gets a sequence number for ordering.

    ✓ More efficient error control
    ✓ Shorter delays
    ✓ Smaller buffers needed
    ✗ Overhead added
    ✗ More processing time
    📥

    Reassembly (at Receiver)

    Counterpart of segmentation — small segments are reassembled back to the original message using sequence numbers.

    Layer ↔ Data Unit
    Application (L5)Message
    Transport (L4)Segment
    Network (L3)Datagram/Packet
    Data Link (L2)Frame
    Physical (L1)Bit

    🔌 Connection Control Methods

    📦

    Connectionless

    Each packet carries the full destination address and is routed independently through the network. The second packet may arrive before the first one.

    UDP Style
    🔗

    Connection-Oriented

    A dedicated path is established before data transfer. Three phases:

    ①Connection EstablishmentRequest → Accept
    ②Data TransferData + ACK
    ③Terminate ConnectionBy any side
    TCP Style

    🔢 Connection Control — Sequencing

    What is Sequencing?

    The key characteristic of connection control. Sequencing numbers outgoing and incoming packets. Supports 3 main functions: Ordered Delivery, Flow Control, and Error Control. Since packets traverse different paths, they may arrive out of order — sequence numbers allow the receiver to re-order them.

    📋

    1. Ordered Delivery

    Used in connectionless (packet switching). Each packet gets a unique sequential number so the receiver can re-order packets that arrived out of sequence.

    🚦

    2. Flow Control

    Performed by receiving entity to limit the amount of data sent by the transmitter. Simplest form: Stop-and-Wait — each PDU must be acknowledged before the next can be sent.

    🛡️

    3. Error Control

    Guards against loss/damage. Steps: ① Sender inserts error-detecting bits → ② Receiver checks → ③ If OK: ACK, if error: discard → ④ No ACK within time: re-transmit.

    🔀 Multiplexing

    Definition

    Multiplexing = set of techniques that allows simultaneous transmission of multiple signals across a single data link. Combines several lower-speed circuits into a higher-speed one. Advantage: cheaper cost — fewer network circuits needed.

    📻 FDM — Frequency-Division Multiplexing

    🎵

    How FDM Works

    FDM divides a large frequency band into smaller independent channels. Each signal is modulated to a different carrier frequency, combined into a composite signal, transmitted, then separated by filters at the receiver.

    1

    Divide bandwidth into smaller frequency channels with guard bands between them

    2

    Modulate each signal onto a different carrier frequency

    3

    Combine all modulated signals into one composite signal (multiplexer)

    4

    Separate at receiver using filters + demodulators (demultiplexer)

    📐

    FDM Example Formulas

    Example 4.2

    5 channels × 100 kHz each + 4 guard bands × 10 kHz =
    5 × 100 + 4 × 10 = 540 kHz minimum link bandwidth

    Guard Bands Formula

    For n channels → need n−1 guard bands.
    Total BW = (n × channel BW) + ((n−1) × guard band)

    FDM was commonly used for telephone signals before digital transmission. Still used on some older lines. Radio/TV stations use FDM with their frequency allocations.

    ⏱️ TDM — Time-Division Multiplexing

    🕐

    Synchronous TDM

    Time on the circuit is shared equally. Each channel gets a fixed time slot whether or not it has data. More efficient than FDM — no guard bands.

    Key Formulas
    Input slot duration = 1 / bit rate
    Output slot duration = input duration / n
    Link data rate = n × connection rate
    Frame duration = = input slot duration

    Example 4.7 — Quick Solve

    4 connections × 1 kbps, unit = 1 bit:
    • Bit duration before MUX = 1 ms
    • Link rate = 4 kbps
    • Time slot = 250 μs
    • Frame = 1 ms

    📊

    Statistical TDM (STDM)

    Time slots are dynamically allocated — only assigned to lines that have data. Number of slots per frame < number of input lines.

    ✓ Advantages
    Higher bandwidth efficiency — idle channels don't waste slots
    ✗ Disadvantages
    • Must add addressing info to each slot (source not identified by position)
    • Response delays if all terminals compete simultaneously
    • Multiplexer needs memory buffer

    ↩️ Inverse Multiplexing

    Works opposite to multiplexing — breaks a higher-speed circuit into two or more lower-speed ones. Used when you need to spread a high-bandwidth connection across multiple lower-bandwidth links.

    📌 6-1 Types of Errors

    ⚡

    Single-Bit Error

    Only 1 bit in the data unit has changed (0→1 or 1→0).

    Sent: 0 0 0 0 0 0 1 0
    Received: 0 0 0 0 1 0 1 0 ← 1 bit flipped
    Least common in real life
    💥

    Burst Error

    2 or more bits have changed. Length = span from first to last corrupted bit (includes correct bits in between).

    🔹 Length of burst = span, NOT count of corrupted bits
    🔹 Most common type in real networks
    Most common

    🔍 Error Detection

    Only check if error occurred. Don't know which bits. Simpler. Used with retransmission (ARQ).

    🔧 Error Correction (FEC)

    Must know exact number + location of corrupted bits. Needs more redundant bits. No retransmission needed.

    🔁 Redundancy — The Key Idea

    To detect/correct errors we must send extra (redundant) bits with the data. The sender creates a relationship between redundant bits and actual data. The receiver checks this relationship.

    📤 Sender
    Message → Generator → Message + Redundancy → send
    📥 Receiver
    Received info → Checker → Correct or Discard → Message

    📌 6-2 Checksum (Internet Checksum) ⭐ EXAM

    What is Checksum?

    Used in the Internet (TCP, UDP, IP) — but NOT at Data Link Layer. Based on one's complement arithmetic.

    📤 Sender — 5 Steps

    1

    Divide message into 16-bit words

    2

    Set checksum word to 0

    3

    Add all words using one's complement addition

    4

    Complement the sum → becomes the Checksum

    5

    Send checksum with data

    📥 Receiver — 4 Steps

    1

    Divide message (including checksum) into 16-bit words

    2

    Add all words using one's complement addition

    3

    Complement the sum

    4

    Result = 0000…0 → No error ✅ Accept
    Result ≠ 0 → Error ❌ Reject

    ⭐ One's Complement — Rules

    🔹 Wrapping: If sum has more bits than word size, wrap leftmost bits and add to rest
    21 = 10101 (5 bits) → 0101 + 1 = 0110 = 6
    🔹 Complement (Negative): Invert ALL bits
    +6 = 0110 → –6 = 1001 = 9

    ⭐ Example 6.22 — Numbers (7, 11, 12, 0, 6)

    Sender Site
    Numbers: 7, 11, 12, 0, 6
    Sum = 36 → Wrapped = 6
    Checksum = 9 (complement of 6 = 1001)
    Sends: 7, 11, 12, 0, 6, 9
    Receiver Site
    Sum all = 7+11+12+0+6+9 = 45
    Wrapped = 15
    Complement of 15 = 0 ✅
    Result = 0 → No Error! Accept ✅

    ⭐ Example 6.23 — "Forouzan" (Internet Checksum — 4 × 16-bit words)

    📤 Sender
    Fo → 4666 6F → 46 6F
    ro → 72 6F
    uz → 75 7A
    an → 61 6E
    ChkSum → 00 00
    ──────────────
    SUM → 8F C7 (after wrap)
    1's Complement → 70 38 ← send this
    📥 Receiver
    46 6F + 72 6F + 75 7A + 61 6E
    + Checksum: 70 38
    ──────────────
    SUM → FF FE + carry 1 = FF FF
    1's Complement → 00 00 ✅
    = 0 → No Errors! Accept ✅

    📌 SNR, Shannon Capacity & Latency ⭐ EXAM

    ⭐ SNR — Signal-to-Noise Ratio

    SNR = Signal Power / Noise Power
    SNR_dB = 10 × log₁₀(SNR)
    🔹 Higher SNR = better signal, less noise, fewer errors
    🔹 SNR is used in Shannon's formula
    🔹 Low SNR → noisy channel → more errors
    🔹 High SNR → clean channel → fewer errors

    ⭐ Shannon Capacity

    C = B × log₂(1 + SNR)
    🔹 C = theoretical max capacity (bps)
    🔹 B = Bandwidth (Hz)
    🔹 SNR = Signal-to-Noise Ratio
    🔹 Cannot exceed C no matter how many signal levels
    🔹 Nyquist = noiseless; Shannon = noisy

    ⭐ Latency (Delay) — Full Formula

    Latency = Propagation Time + Transmission Time + Queuing Time + Processing Time
    📡 Propagation Time
    = Distance / Propagation Speed
    Time for signal to travel through medium
    📤 Transmission Time
    = Message Size (bits) / Bandwidth
    Time to push all bits onto wire
    ⏳ Queuing Time
    Waiting time in queue at intermediate devices
    ⚙️ Processing Time
    Time for routers to process headers

    💡 Propagation Speed Values

    Vacuum/Air: 3 × 10⁸ m/s  |  Copper: 2.3 × 10⁸ m/s  |  Fiber: 2 × 10⁸ m/s

    ⭐ CSMA/CD — How it Works

    1

    Carrier Sense: Listen to channel first. If busy → wait. If idle → transmit.

    2

    3

    📌 CRC — Cyclic Redundancy Check ⭐ EXAM

    Used at Data Link Layer. More powerful than checksum. Based on binary division (modulo-2 / XOR arithmetic).

    📤 Sender — CRC
    1. Take message (dataword)
    2. Append zeros = degree of generator
    3. Divide by generator using XOR
    4. Remainder = CRC (check bits)
    5. Send: message + CRC = codeword
    📥 Receiver — CRC Check
    1. Receive codeword
    2. Divide by same generator using XOR
    3. Remainder = 0 → No error ✅
    4. Remainder ≠ 0 → Error ❌ Reject

    💡 CRC vs Checksum

    🔹 CRC detects all single-bit errors & burst errors shorter than generator
    🔹 CRC used in: Ethernet, Wi-Fi, USB, storage
    🔹 Checksum simpler but weaker — used in IP/TCP/UDP headers

    📌 MAC Addressing ⭐ EXAM

    Each station on an Ethernet network has its own NIC (Network Interface Card). The NIC has a unique 6-byte (48-bit) physical MAC address written in hexadecimal.

    06 : 01 : 02 : 01 : 2C : 4B
    6 bytes = 12 hex digits = 48 bits
    🟢 Unicast

    Only one recipient. Least Significant Bit (LSB) of first byte = 0.
    Used for source addresses.
    Example: 4A:30:10:21:10:1A
    "A"=1010 → even → unicast ✅

    🟠 Multicast

    Recipients are a group of stations. LSB of first byte = 1.
    Example: 47:20:1B:2E:08:EE
    "7"=0111 → odd → multicast ✅

    🔴 Broadcast

    Recipients are ALL stations on the LAN. All 48 bits = 1s (special case of multicast).
    Address: FF:FF:FF:FF:FF:FF

    ⭐ How to Determine Address Type — Exam Rule

    Look at the Least Significant Bit (LSB) of the FIRST byte of the address:

    0 = Even → Unicast address 4A → A=1010 (even) → Unicast
    1 = Odd → Multicast address 47 → 7=0111 (odd) → Multicast
    FF = All 1s → Broadcast address FF:FF:FF:FF:FF:FF

    💡 Transmission Order — Example 7.2

    Address is sent left-to-right, byte by byte; for each byte, it is sent right-to-left, bit by bit.
    Address 47:20:1B:2E:08:EE → sent as: 11100010 00000100 11011000 01110100 00010000 01110111

    📌 CSMA/CD — Full Algorithm ⭐ EXAM

    How CSMA/CD Works — 4 Key Points

    1

    Monitor: The computer monitors the network (Carrier Sense). Only one computer can transmit data at the same time.

    2

    Transmit when idle: It transmits when it senses that the cable is not busy with any signal. Listen before sending.

    3

    Collision: If a collision occurs because another computer is sending at the same time → both computers stop sending immediately and broadcast a Jam signal.

    4

    Back-off: Each station waits a random time (binary exponential back-off: choose R between 0 and 2ᴷ−1, wait R×Tp) to retransmit, reducing the possibility of another collision. K_max = 15.

    💡 Key CSMA/CD Facts for Exam

    🔹 Works only in Half-Duplex (shared medium)
    🔹 Modern Switched Ethernet = Full-Duplex → No collisions → CSMA/CD not needed
    🔹 Min frame = 64 bytes (so sender is still transmitting when collision signal arrives back)
    🔹 Max connections per Ethernet network: 1024 (without bridges/routers)
    🔹 Each NIC has an Ethernet Network Controller that monitors whether cable is free

    📌 7-3 Changes in the Standard ⭐ EXAM

    🌉 Bridged Ethernet

    Divides a crowded network into two collision domains using a Bridge. Reduces congestion — fewer users compete on each segment. Useful when:
    🔹 Many new users join
    🔹 High-bandwidth apps needed (databases, video)

    🔀 Switched Ethernet

    Each station connects to a central switch. Each connection is its own collision domain (1 station = 1 domain). Switch forwards frames only to the correct destination port. Much more efficient than shared bus.

    ⚡ Full-Duplex Ethernet

    With a switch, stations can transmit and receive simultaneously — no collisions possible. CSMA/CD is not needed. Doubles effective bandwidth compared to half-duplex.

    ◀▶ Half-Duplex
    🔹 Stations connected via Hub
    🔹 One direction at a time
    🔹 CSMA/CD required
    🔹 Collisions possible
    🔹 Used in older Standard Ethernet bus topology
    ◀▶◀▶ Full-Duplex
    🔹 Connected via Switch with buffer per port
    🔹 Both directions simultaneously
    🔹 NO CSMA/CD needed
    🔹 NO collisions
    🔹 Used in Fast, Gigabit, 10-Gigabit Ethernet

    📌 7-4 Fast Ethernet (100 Mbps — IEEE 802.3u)

    Fast Ethernet transmits data 10× faster than Standard Ethernet at 100 Mbps. Backward-compatible with Standard Ethernet. Drops Bus topology, keeps Star. Supports auto-negotiation.

    1000 Mbps. Supports both Half-duplex (hub, CSMA/CD) and Full-duplex (switch, no CSMA/CD). In full-duplex mode: no collision, max length determined by signal attenuation.
