Introduction to HTML
What HTML is — and what it isn't

What is HTML?

  • Stands for HyperText Markup Language
  • Describes the structure of a web page
  • Consists of a series of elements
  • Elements label pieces of content ("this is a heading", "this is a link")
  • Follows a "living standard" — always updated, features are never removed

Markup vs. Programming

MarkupProgramming
Presents dataTransforms data
No logical opsLogical operations
Parsed by browserCompiled + interpreted
HTML, XMLC++, Java, C#

💡 Weird Browser Fact

Browsers never reject invalid HTML — they silently try to render it. This is by design: the web grew too big to enforce strict rules.

The skeleton every page is built on
📄 Basic Page Structure 
<!DOCTYPE html>
<html lang="en">

  <head>
    <meta charset="utf-8">
    <title>Page Title</title>       <!-- shown in browser tab -->

  <body>

  </body>

Start & End Tags

Most elements have an opening and closing tag:

Block vs. Inline Elements
How elements flow in the document

Block-Level Elements

  • Always start on a new line
  • Take full width available (100%)
  • Respect width & height
  • Examples: <p> <div> <h1>..<h6> <ul> <table>

Inline Elements

  • Do not start on a new line
  • Only take as much width as needed
  • Do NOT respect width & height
  • Examples: <a> <span> <strong> <em>

The <iframe> Element

<iframe src="https://www.example.com" width="600" height="400" title="Example">
</iframe>

HTML Forms — Overview
Collecting user data — the three core elements

What is an HTML Form?

  • Used to collect user input
  • User input is most often sent to a server for processing
  • Three core building blocks: <form>, <label>, <input>

The <form> Element

A container for all types of input elements:

  • Text fields
  • Checkboxes
  • Radio buttons
  • Submit buttons, etc.
<form>
  . form elements .
</form>

The <label> Element

  • Defines a label for form elements
  • Essential for screen-reader users — reads aloud when input is focused
  • Clicking label toggles associated radio/checkbox
  • for attribute must equal the input's id
<label for="html">HTML</label>
<input type="radio" id="html" name="fav">

11
HTML Form Attributes
action · target · method

action Attribute

Defines the action to be performed when the form is submitted. Usually the form data is sent to a file on the server when the user clicks the submit button.

<form action="/action_page.php">
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" name="fname" value="John"><br>
  <label for="lname">Last name:</label><br>
  <input type="text" id="lname" name="lname" value="Doe"><br><br>
  <input type="submit" value="Submit">
</form>

target Attribute

Specifies where to display the response received after submitting the form.

ValueDescription
_blankResponse displayed in a new window or tab
_selfResponse displayed in the current window (default)
_parentResponse displayed in the parent frame
_topResponse displayed in the full body of the window
framenameResponse displayed in a named iframe
 
<form action="/action_page.php" target="_blank">
  <!-- Opens result in a new tab -->
</form>

method Attribute

Specifies the HTTP method to use when submitting. Default is GET.

method="get"

Form data appended to URL as variables. Values visible in address bar. Use for non-sensitive data.

method="post"

Form data sent as HTTP transaction body. Values NOT visible in URL. Use for sensitive data.

<form action="/action_page.php" target="_blank" method="get">
  <!-- After submit, values visible in URL bar -->
</form>

HTML Input Attributes
Fine-tuning input element behavior

All Input Attributes Overview

The <input> element is the most-used form element. Its behavior changes based on the type attribute and the following additional attributes:

disabled size maxlength min / max multiple pattern placeholder required step autofocus height / width list autocomplete

disabled

Makes an input field unusable and unclickable. Its value will not be submitted.

<input type="text" name="fname"
       value="John" disabled>

size

Specifies the visible width in characters. Default is 20. Works with: text, search, tel, url, email, password.

<input type="text" name="fname" size="50">
<input type="text" name="pin" size="4">

maxlength

Specifies the maximum number of characters allowed. Does not provide feedback — use JavaScript for alerts.

<input type="text" name="pin"
       maxlength="4" size="4">

min and max

Specify minimum and maximum values. Use together to create a range of legal values. Works with: number, range, date, datetime-local, month, time, week.

<input type="date" name="bday"
       max="1979-12-31">
<input type="number" name="qty"
       min="1" max="5">

multiple

Allows user to enter more than one value. Works with: email, file.

<input type="file" name="files"
       multiple>
<!-- Hold CTRL/SHIFT to multi-select -->

pattern

Specifies a regular expression the field value is checked against on submit. Works with: text, date, search, url, tel, email, password.

<input type="text" name="country_code"
  pattern="[A-Za-z]{3}"
  title="Three letter country code">
Tip: Use the title attribute to describe the pattern to the user.

placeholder

Shows a short hint describing the expected input value. Disappears when user starts typing. Works with: text, search, url, number, tel, email, password.

<input type="tel" name="phone"
  placeholder="123-45-678"
  pattern="[0-9]{3}-[0-9]{2}-[0-9]{3}">

required

Specifies that an input must be filled out before submitting. Works with: text, search, url, tel, email, password, date pickers, number, checkbox, radio, file.

<input type="text" name="username"
       required>

step

Specifies the legal number intervals. E.g., step="3" → legal values: -3, 0, 3, 6… Can be combined with min/max. Works with: number, range, date, datetime-local, month, time, week.

<input type="number" name="points"
       step="3">

autofocus

The input field automatically gets focus when the page loads.

<input type="text" name="fname"
       autofocus>

height and width

Specifies dimensions for <input type="image">. Always specify both so the browser can reserve space during loading.

<input type="image" src="submit.gif"
  alt="Submit" width="48" height="48">

list

Refers to a <datalist> containing predefined options for autocomplete suggestions.

<input type="text" name="browser"
       list="browsers">
<datalist id="browsers">
  <option value="Google Chrome">
  <option value="Mozilla Firefox">
  <option value="Microsoft Edge">
</datalist>

autocomplete

Specifies whether the browser should predict/autocomplete values based on previously typed entries. Can be set on the whole <form> or individual <input> elements. Works with: text, search, url, tel, email, password, datepickers, range, color.

<form action="/action_page.php" autocomplete="on">
  <input type="text" name="fname">
  <!-- Form has autocomplete on... -->
  <input type="email" name="email" autocomplete="off">
  <!-- ...but this specific field has it off -->
</form>

MVC Development Model
Model · View · Controller — architectural pattern for web apps

What is MVC?

The MVC architecture is a widely used approach for interactive applications that distributes functionality among application objects so as to minimize the degree of coupling between the objects.

It divides an application into three layers, each with specific responsibilities:

View — Presentation Layer Controller — Interface Layer Model — Data Layer

The Three Layers

🖥️ View Layer

  • Represents the presentation layer (User Interface)
  • Normally represents the UI of the application
  • HTML, CSS, JS etc. are used to create the View

⚙️ Controller Layer

  • Acts as an interface between View and Model
  • Receives requests from the View layer
  • Reads data coming from the presentation layer and forwards it to the Model

🗄️ Model Layer

  • Is the data layer — contains all data of the web application
  • Represents the current state of the application
  • Responsible for connecting with the database and storing data

Why MVC Reduces Coupling

Each component has a single, clear responsibility:

  • You can change the View (UI design) without changing the Model
  • You can change the Model (business rules) without touching the View
  • The Controller coordinates without tightly binding components together
  • Each object interacts through well-defined interfaces, not internal details

MVC Request Flow — Login Example

Trace how a login form flows through the MVC layers:

1
login.php — User clicks a link or submits a form View
2
Loginphp.php — A Servlet receives the request Controller
3
User.php — The Servlet uses this class to get or update data Model
4
If credentials valid → forward to welcome.php View
5
If invalid → redirect to error.php View

MVC Code Example

View — login.php 
<form action="LoginServlet" method="post">
  Username:
  <input type="text" name="username"/>
  Password:
  <input type="password" name="password"/>
  <input type="submit" value="Login"/>
</form>
Model — User.php 
public class User {
  private String username, password;

  public String getName() {
    return username;
  }
  public boolean isValid() {
    return username.equals("admin") &&
           password.equals("1234");
  }
}

Advantages of MVC Architecture

  • ✅ Easy to maintain
  • ✅ Easy to test
  • ✅ Easy to extend
  • ✅ Navigation control is centralized
