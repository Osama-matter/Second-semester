🔥 Ternary Trap — Chained Order
let score = 85;
let grade = score >= 90 ? 'A' :
            score >= 70 ? 'C' :  // ← TRAP!
            score >= 80 ? 'B' :
            score >= 60 ? 'D' : 'F';
// Answer: 'C' NOT 'B'!
// 85 >= 90? No. 85 >= 70? YES → 'C'
// The B check never runs!
Ternary evaluates LEFT TO RIGHT! Order of conditions matters. >=70 matches before >=80 even if score is 85.

🍪 Cookie Code Patterns
<?php
// SET a cookie (expires in 1 hour)
setcookie("name", "John", time()+3600, "/");
setcookie("age", "36", time()+3600, "/");

// READ a cookie
echo $_COOKIE["name"];  // John

// CHECK if cookie exists
if (isset($_COOKIE["name"])) {
  echo "Welcome " . $_COOKIE["name"];
} else {
  echo "Not recognized";
}

// DELETE a cookie (set past expiry)
setcookie("name", "", time()-60, "/");
?>
⚠ setcookie() MUST be before ANY HTML output including <html>!
Each cookie needs a separate setcookie() call.

🔥 session_destroy vs unset
  • session_destroy() = removes ALL session data + SESSID cookie
  • unset($_SESSION['key']) = removes ONE variable; rest stays
  • Always call session_unset() BEFORE session_destroy() for clean logout
Session Flow:
session_start() → set $_SESSION → access on other pages (call session_start first) → logout: session_unset() + session_destroy()
⚙ How Sessions Work Internally
  • session_start() creates a unique session ID
  • Session ID stored as cookie named PHPSESSID in browser
  • Actual data stored in server's temp files
  • Browser sends PHPSESSID on every request
  • PHP finds and loads data from server storage
  • Sessions work even if cookies are blocked (URL rewriting)

📖 Reading Functions — Which to Use?

✍ Writing Functions
// HIGH-LEVEL (simpler)
file_put_contents('file.txt', 'Hello');        // overwrite
file_put_contents('log.txt', $entry, FILE_APPEND);    // append
file_put_contents('log.txt', $entry, FILE_APPEND | LOCK_EX); // safe

// LOW-LEVEL (more control)
$handle = fopen('export.csv', 'w');
if ($handle === false) exit('Cannot open');

fwrite($handle, "id,name\n");  // write header
foreach ($users as $u) {
  fwrite($handle, $u['id'].','.$u['name']."\n");
}
fclose($handle); // ← NEVER forget this!
⚠ Always check return value! file_put_contents() returns false on error (disk full, permissions). Never assume success!
📁 Reading Patterns
// readfile() — download

// file_get_contents — manipulate

$handle = fopen('users.csv', 'r');
while (($line = fgets($handle)) !== false) {
  $row = str_getcsv(trim($line));
  // process $row...
}
fclose($handle);

// Best practice pattern:
$handle = fopen($path, 'r');
if ($handle === false) throw new Exception("Cannot open");
try {
  while (($line = fgets($handle)) !== false) {
    processLine(trim($line));
  }
} finally {
  fclose($handle); // runs even if error!
}

🔥 LAST-MINUTE MUST-KNOW — TOP 15 EXAM TRAPS
1. typeof null = "object" — JavaScript bug, kept for compatibility
2. PHP uses . (dot) for concat, NOT + — JavaScript uses +
3. GET = visible in URL | POST = hidden in body — POST is safer for passwords
4. disabled fields are NOT submitted | readonly fields ARE submitted
5. Cookies stored on CLIENT | Sessions stored on SERVER
6. setcookie() must be before ANY HTML output (including <html>)
7. let = block scope | var = function scope
8. Ternary evaluates left→right — >=70 matches before >=80 if score is 85!
9. 5 + '5' + 1 = "551" — once string appears, + becomes concat
10. fopen('file','w') IMMEDIATELY erases the file — even before writing!
11. scandir() always returns . and .. — filter with array_diff
12. Inline style > ID > Class > Element — specificity order
13. align-items = cross axis | justify-content = main axis in flexbox
14. PHP function names are NOT case-sensitive | variables ARE
15. do...while always runs at least once — condition checked AFTER body
