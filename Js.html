Module 01

JS Foundations

What JavaScript is, how it runs, where it lives in HTML, and how to output data to the browser.

📌 Key Concepts

  • Scripting language — enhances functionality & appearance of web pages
  • Client-side — runs in the browser, not on the server
  • Dynamic & lightweight — interpreted at runtime (not compiled)
  • Object-oriented — supports OOP capabilities
  • Case-sensitive — myVar ≠ MyVar
  • Semicolons — best practice to terminate statements with ;

🏆 4 Advantages

  • Less server interaction — validate input before sending to server
  • Immediate feedback — no page reload needed
  • Increased interactivity — hover, keyboard, drag-and-drop
  • Richer interfaces — sliders, accordions, dynamic UIs

Output Methods

📤 4 Output Ways

  • innerHTML / innerText — writes into an element (most common)
  • document.write() — writes to HTML stream (test only!)
  • window.alert() — popup alert box
  • console.log() — browser DevTools console (debugging)

💡 innerHTML vs innerText

  • innerHTML — parses HTML tags inside the string
  • innerText — treats content as plain text, no HTML parsing
example.js
document.getElementById("demo").innerHTML = "<b>Bold!</b>";
document.getElementById("demo").innerText = "Plain text";
⚠️ CRITICAL WARNING
  • Using document.write() after the page has fully loaded will erase all existing HTML on the page.
  • Never use document.write() in production — it is for quick testing only.
  • The window keyword is optional: alert() === window.alert()

⚠️ CRITICAL — const Arrays & Objects

const does NOT make the value immutable — it locks the reference.

  • You CAN change array elements: cars[0] = "Toyota" ✅
  • You CAN add properties to an object: car.owner = "Ahmed" ✅
  • You CANNOT reassign the variable: cars = [...] ❌
  • You CANNOT reassign the object: car = {...} ❌
variables.js
// var — global scope, can redeclare
var x = 10;
{
  var x = 2; // same x globally — x is now 2
}

// let — block scope
let y = 10;
{
  let y = 2; // different y — block scoped
}
// y is still 10 here

// const — block scope, no reassign
const PI = 3.14159;
PI = 3.14; // TypeError!

Module 03

Operators & Control Flow

All operator types, if/else, switch, and the ternary operator for conditional logic.

Operator Quick Reference

➕ Arithmetic

  • + Addition
  • - Subtraction
  • * Multiplication
  • ** Exponentiation (ES6)
  • / Division
  • % Modulus (remainder)
  • ++ Increment  /  -- Decrement

📊 Comparison

  • == Equal value (type coercion)
  • === Equal value AND type (strict)
  • != Not equal
  • !== Not equal value or type
  • > < >= <= Greater/less than

🔗 Logical

  • && AND — both operands must be truthy
  • || OR — at least one must be truthy
  • ! NOT — reverses boolean state

🔄 Assignment

  • x = y   x += y   x -= y
  • x *= y  x /= y  x %= y
  • x **= y — exponent assign
⚠️ == vs === — EXAM FAVOURITE
  • 5 == '5' → true (string '5' coerced to number)
  • 5 === '5' → false (number ≠ string type)
  • null == undefined → true (special JS case)
  • Always prefer === to avoid unexpected coercion bugs
Ternary Operator
ternary.js
// Syntax: condition ? valueIfTrue : valueIfFalse
let age = 26;
let drink = (age >= 21) ? "Beer" : "Juice";

// Chained ternary (right-associative)
let score = 85;
let grade = score >= 90 ? 'A'
           : score >= 80 ? 'B'
           : score >= 70 ? 'C'
           : score >= 60 ? 'D' : 'F'; // → 'B'
if / else if / else & switch
control-flow.js
// if…else
if (num1 > num2 && num1 > num3) {
  document.write("num1 is largest");
} else if (num2 > num1 && num2 > num3) {
  document.write("num2 is largest");
} else {
  window.alert("All numbers are equal");
}

// switch
switch(pwd) {
  case "admin":
    document.write("Authorised");
    break;
  default:
    window.alert("Unauthorised");
}
💡 switch vs if-else

Use switch when all branches depend on a single variable's value — it is more efficient and readable than repeated if…else if chains.

Module 04

Functions & Loops

Defining and calling functions, built-in functions, and the three loop structures with flow diagrams.

🔧 Functions — Key Facts

  • Block of code designed to perform a specific task
  • Reusable — eliminates code duplication
  • Defined with function keyword + name + (params) + { body }
  • return passes value back to the caller
  • Called explicitly: just write functionName()

🔁 3 Loop Types

  • while — tests condition BEFORE executing body
  • for — init; condition; iteration in one line
  • do…while — tests AFTER body — executes AT LEAST ONCE
Function Syntax
functions.js
// Declaration
function square(j) {
  return j * j;
}

// Call it
for (var i = 1; i <= 10; i++) {
  document.write("Square of " + i + " = " + square(i));
}

// Using Math built-in functions
let maxVal = Math.max(x, Math.max(y, z));
let parsed = parseFloat(prompt("Enter a number", 0.0));
Loop Structures
loops.js
// ── while loop ──────────────────────────────
let i = 1;
while (i <= 10) {
  document.write(i);
  i++;
}

// ── for loop ────────────────────────────────
let fact = 1;
for (var i = 1; i <= n; i++) {
  fact = fact * i; // factorial
}

// ── do…while loop ───────────────────────────
var counter = 1;
do {
  document.writeln("<h" + counter + ">heading</h" + counter + ">");
  ++counter;
} while (counter <= 6);
⚠️ do…while WARNING

The do…while loop body always runs at least once, even if the condition is false from the start. This is the key difference from a regular while loop.

Module 05

DOM & Events

Manipulating HTML with JavaScript, the Window object, event handling with addEventListener.

🌳 DOM Concepts

  • DOM = Document Object Model (tree-like structure)
  • document = root node of HTML document
  • Access via document.getElementById(id)
  • Manipulate: content, attributes, style, visibility
  • document is a property of window

🪟 Window Object

  • Represents the open browser window
  • window.alert() — popup message
  • window.confirm() — OK/Cancel dialog
  • window.prompt() — input dialog
  • window.print() — print current page
  • window.stop() — stop page loading

Event Handling
events.js
// Inline HTML event attribute
// <button onclick="myFunction()">Click Me</button>

// document.addEventListener — attaches to whole document
document.addEventListener("click", function() {
  document.getElementById("demo").innerHTML = "Clicked!";
});

// element.addEventListener — attaches to specific element
const btn = document.getElementById("myBtn");
btn.addEventListener("mouseover", myFunction);
btn.addEventListener("click", mySecondFunction);
btn.addEventListener("mouseout", myThirdFunction);

// Multiple events on SAME element is allowed!
Common Events

🖱️ Mouse Events

  • click — user clicks element
  • dblclick — double click
  • mousedown — press mouse button
  • mouseup — release mouse button
  • mousemove — mouse moves over element
  • mouseover — pointer enters element
  • mouseout — pointer leaves element

🪟 Window Events

  • load — whole page has loaded
  • resize — browser window resized
  • scroll — user scrolls element

⌨️ Form / Key Events

  • change — input field changed
  • submit — form submitted
  • keydown — key pressed
⚠️ addEventListener SYNTAX — Don't confuse these
  • element.addEventListener("click", myFn) ← Correct — pass reference
  • element.addEventListener("click", myFn()) ← Wrong — this calls the function immediately, not on click
  • Use anonymous arrow syntax as alternative: element.addEventListener("click", () => { ... })
Prompt + Switch Pattern (Common Exam Question)
prompt-switch.js
let favDrink = prompt("Favourite drink?", "Coca-Cola");
switch(favDrink) {
  case "Coca-Cola":
    document.getElementById("demo").innerHTML = "Great choice!";
    break;
  case "Pepsi":
    document.getElementById("demo").innerHTML = "My fav too!";
    break;
  default:
    document.getElementById("demo").innerHTML = "Never heard of it";
}
